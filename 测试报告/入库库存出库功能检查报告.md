# 入库、库存、出库 功能完整性检查报告

## 一、结论概览

| 模块 | 后端 | 前端 | 结论 |
|------|------|------|------|
| **入库** | 服务与模型存在，但未接入 DB/API/DI，且业务不完整 | 无页面、无路由、无接口 | **不完整** |
| **库存** | 仅有模型，无服务与 API | 仅有菜单入口，无路由与页面 | **不完整** |
| **出库** | 同上，服务存在但未接入且业务不完整 | 无页面、无路由、无接口 | **不完整** |

## 二、后端检查

### 2.1 数据层（DbContext）

**问题：入库/出库/库存相关表未在 DbContext 中注册。**

- **位置**：`CRM.Infrastructure/Data/ApplicationDbContext.cs`
- **现状**：
  - 已注册：Users, Customers, Vendor*, SerialNumbers, ErrorLogs, ComponentCaches 等
  - **未注册**：`StockInfo`、`StockIn`、`StockInItem`、`StockOut`、`StockOutItem`、`StockOutRequest`
- **影响**：EF 不会为这些实体建表/映射，`IRepository<StockIn>` 等即使注入也会在运行时因 `context.Set<StockIn>()` 无法正确映射而出错或无法持久化。

**需补充**：在 `ApplicationDbContext` 中增加上述实体的 `DbSet`，并在 `OnModelCreating` 中做必要配置（主键、长度、关系等）。

---

## 三、前端检查

### 3.1 路由

**位置**：`CRM.Web/src/router/routes.ts`

- **现状**：仅有 Dashboard、客户、供应商、系统设置等路由。
- **缺失**：
  - `/inventory/list`（库存列表）
  - `/inventory/transfers-manual`（手工移库）
  - `/inventory/check`（库存盘点）
  - 入库相关路由（如 `/inventory/stock-in` 或 `/stock-in/list`）
  - 出库相关路由（如 `/inventory/stock-out` 或 `/stock-out/list`）

侧栏菜单（`AppLayout.vue`）已存在「库存管理」及子项「库存列表」「库存调拨」「库存盘点」，但上述路径未在路由中配置，点击会 404。

### 3.2 页面与 API

- **入库**：无任何入库列表/新建/编辑页面，无 `api/stockIn.ts` 或等价接口封装。
- **出库**：无出库列表/执行出库等页面，无 `api/stockOut.ts`。
- **库存**：无库存列表、调拨、盘点页面，无 `api/inventory.ts` 或 `api/stock.ts`。

## 四、问题汇总与修复建议

### 4.1 后端必须修复项

1. **DbContext**
   - 增加 `DbSet<StockInfo>`、`DbSet<StockIn>`、`DbSet<StockInItem>`、`DbSet<StockOut>`、`DbSet<StockOutItem>`、`DbSet<StockOutRequest>`。
   - 在 `OnModelCreating` 中配置主键、字段长度、外键关系（如 `StockInItem.StockInId` → `StockIn`）。
   - 生成并执行迁移，确保库中存在对应表。

2. **入库服务**
   - 在 `StockInService` 中注入 `IUnitOfWork`，在所有增删改后调用 `await _unitOfWork.SaveChangesAsync()`。
   - 在 `CreateAsync` 中根据 `request.Items` 创建并保存 `StockInItem`（可再注入 `IRepository<StockInItem>` 或通过 `StockIn.Items` 一并添加后 SaveChanges）。

3. **出库服务**
   - 在 `StockOutService` 中注入 `IUnitOfWork`，在 Add/Update 后调用 `SaveChangesAsync()`。
   - 在 `ExecuteStockOutAsync` 中根据 `request.Items` 创建并保存 `StockOutItem`。

4. **DI 注册**
   - 在 `ServiceExtensions.cs` 中注册：
     - `services.AddScoped<IStockInService, StockInService>();`
     - `services.AddScoped<IStockOutService, StockOutService>();`

5. **API 控制器**
   - 新增 `StockInController`：入库单列表、详情、创建、更新、删除、更新状态等（与现有 `ApiResponse<T>` 风格一致）。
   - 新增 `StockOutController` 或 `StockOutRequestsController`：出库申请创建、出库执行、列表、详情等。
   - （可选）为入库/出库单提供分页、筛选参数。

6. **库存模块**
   - 定义 `IStockService` / `IInventoryService`：库存列表（按仓库/物料）、库存调整、盘点单创建与确认等。
   - 实现服务并注册 DI，新增 `StockController` 或 `InventoryController`，提供库存查询、调拨、盘点等接口。

### 4.2 前端必须修复项

1. **路由**
   - 在 `routes.ts` 中为「库存管理」增加子路由，例如：
     - `path: 'inventory/list'` → 库存列表页
     - `path: 'inventory/transfers-manual'` → 手工移库页
     - `path: 'inventory/check'` → 库存盘点页
   - 如需独立菜单，可再增加入库、出库路由（如 `stock-in/list`、`stock-out/list`）。

2. **页面**
   - 库存列表：表格展示仓库、物料、数量、可用/锁定等，支持筛选。
   - 库存调拨：选择源/目标仓库与物料、数量，调用调拨接口。
   - 库存盘点：盘点单列表、创建、录入实盘数量、确认调整。
   - 入库：入库单列表、新建入库单（表头 + 明细行）、状态更新。
   - 出库：出库申请列表、执行出库（选择申请单生成出库单并填写明细）。

3. **前端 API**
   - 新增 `api/stockIn.ts`：调用入库相关后端接口。
   - 新增 `api/stockOut.ts`：调用出库申请、执行出库等接口。
   - 新增 `api/inventory.ts` 或 `api/stock.ts`：库存列表、调拨、盘点等接口。
   - 与现有 `apiClient`、`ApiResponse` 解包方式保持一致。

---

## 五、小结

- **入库、库存、出库** 在前后端均**未完整实现**：模型与部分服务存在，但未接入数据库、未暴露 API、未注册 DI，且入库/出库服务未落库（无 UnitOfWork）、未保存明细行；库存仅有模型，无服务与接口。
- 前端仅有库存管理菜单入口，无对应路由与页面，也无入库/出库/库存的接口封装。
- 按上述「后端必须修复项」与「前端必须修复项」逐项实现后，入库、库存、出库功能方可完整可用。
