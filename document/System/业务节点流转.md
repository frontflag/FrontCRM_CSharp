# 业务节点流转：从需求到销售收款完成

## 1. 文档目的与适用范围

本文档用**一条端到端业务链**串联：**需求（RFQ）→ 报价 → 销售订单 → 以销定采与出入库 → 应收与（可选）开票 → 收款核销直至收款完成**。

- **实现与规则细节**以各专题文档及 `CRM.Core` 服务层为准；本文只做**节点级**索引，避免与专题文档重复维护同一套细则。
- **采购侧付款**（对供应商）不在本文主链内；若需对照，见 [收付款与发票业务逻辑](../EBS/财务/收付款与发票业务逻辑.md) 与 [业务规则总览](./业务规则总览.md) §10。

---

## 2. 端到端总览

```text
需求(RFQ) → 报价(Quote) → 销售订单 → [销售审核/履约状态]
    → 采购申请 → 采购订单 → 采购入库 → 出库申请 → 执行出库
    → 生成应收 → [可选：申请开票 / 销项发票]
    → 收款单(草稿→审→确认收款) → 核销应收 → 收款完成(回写订单/发票状态)
```

与系统数据流表述一致的一段子链（从销售订单起）见 [业务逻辑与数据关系文档](./业务逻辑与数据关系文档.md) **§7.3.1**：

```text
创建销售订单 → 生成采购申请 → 创建采购订单 → 入库 → 出库 → 收款开票
```

---

## 3. 分阶段业务节点

### 3.1 售前：需求与报价

| 序号 | 业务节点 | 说明 |
|------|----------|------|
| 1 | **创建需求（RFQ）** | 客户、明细（如 MPN/品牌）等；单号服务端生成。 |
| 2 | **需求处理 / 分配采购员** | 轮询或手工分配等（依实现）。 |
| 3 | **创建报价（Quote）** | 可与 `RFQItemId` 绑定；创建后可能回写需求明细状态。 |

**参考**：[业务规则总览](./业务规则总览.md) §3 需求、§4 报价；需求主状态枚举见 [RFQ 主状态枚举](./RFQ主状态枚举.md)。

---

### 3.2 销售订单：建档、审核、履约入口

| 序号 | 业务节点 | 说明 |
|------|----------|------|
| 4 | **创建销售订单** | 主表、明细、`SellOrderItemExtend` 初始化开票/收款等额度。 |
| 5 | **提交审核 / 审核** | 主状态：`New → PendingAudit → Approved`（或审核失败等）；通过后满足出库申请等前置条件之一。 |
| 6 | **申请出库（主单维度）** | 可将主单置为进行中（`InProgress`），与 [业务规则总览](./业务规则总览.md) §5 一致。 |

**参考**：销售主状态枚举见仓库内 `SellOrderMainStatus` 与 [销售订单主状态](./销售订单主状态.md)（若存在）；扩展字段见 [SellOrderItemExtend 统计字段文档](../EBS/销售/SellOrderItemExtend统计字段文档.md)；明细编号方案见 [销售订单明细编号](../实现方案/销售订单明细编号.md)。

---

### 3.3 以销定采与库存

| 序号 | 业务节点 | 说明 |
|------|----------|------|
| 7 | **采购申请（PR）** | 由销售明细驱动；数量占用与 `remainingQty` 等规则见专题。 |
| 8 | **采购订单（PO）** | 创建、审核、确认、执行；可从销售自动生成或关联 `SellOrderItemId`。 |
| 9 | **采购入库** | 入仓、回写采购与库存；单价等规则见入库服务与物流文档。 |
| 10 | **出库申请（出库通知）** | 主单已审且非完结等；同一销售行多次申请时累计不超过销售数量。 |
| 11 | **拣货与执行出库** | 关联申请、拣货完成后扣减库存。 |

**参考**：[业务规则总览](./业务规则总览.md) §6 采购订单、§7 采购申请、§8 出库、§9 入库；[入库库存出库业务关系文档](../EBS/物流/入库库存出库业务关系文档.md)；[申请出库](../EBS/物流/申请出库.md)；[库存设计与实现方案（StockItem 版）](./库存设计与实现方案-StockItem版.md)；采购扩展字段 [PurchaseOrderItemExtend 统计字段文档](../EBS/采购/PurchaseOrderItemExtend统计字段文档.md)。

---

### 3.4 应收、开票（可选）、收款完成

| 序号 | 业务节点 | 说明 |
|------|----------|------|
| 12 | **生成应收** | 文档口径：**销售出库完成后**产生应收待核销数据，并与销售订单/装箱等关联。 |
| 13 | **（可选）申请开票 → 开票通知 → 销项发票** | 税务与收入确认路径；可与收款状态联动，见财务专题。 |
| 14 | **创建收款单** | 财务 Phase1：`FinanceReceipt` 从草稿起。 |
| 15 | **收款单审核** | `草稿(0) → 待审核(1) → 已审核(2)`。 |
| 16 | **确认收款并核销应收** | `已审核(2) → 已收款(3)`；核销金额与待核销余额校验。 |
| 17 | **收款完成** | 收款单达「已收款」；应收冲抵完毕；回写销项发票收款状态、销售订单 `finance_receipt_status` 及明细扩展收款进度等。 |

**参考**：[收付款与发票业务逻辑](../EBS/财务/收付款与发票业务逻辑.md)（§1.3 整体流程、§3.1 收款流程、§9.1 流程总览）；[财务 Phase1 状态流转矩阵](../PRD/辅助功能/财务Phase1状态流转矩阵.md) §2 收款单；收款进度同步示例见 [业务逻辑与数据关系文档](./业务逻辑与数据关系文档.md) **§7.3.2**。

## 4. 变体与说明

| 场景 | 说明 |
|------|------|
| **现货直出** | 若不经由采购补货，则节点 7～9 可能缩短或跳过，仍以**出库完成 → 应收**为财务前置口径（见财务文档）。 |
| **发票** | 节点 13 为可选分支；最短收款路径可为：应收 → 收款单 → 核销 → 完成。 |
| **取消 / 异常** | 各单据负向状态（取消、驳回等）以 [财务 Phase1 状态流转矩阵](../PRD/辅助功能/财务Phase1状态流转矩阵.md) 与各服务状态机为准。 |
