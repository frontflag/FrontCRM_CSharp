# 系统设计规范

本文档约定系统级设计与实现原则，与具体模块 PRD、实现方案配合使用。

---

## 数据持久化与编辑界面

**除非经产品/架构书面明确同意，禁止通过「将多个编辑界面字段拼接进单一 `comment` / 备注类文本字段」的方式持久化保存。**

说明：

- 包括但不限于：用固定前缀 + 换行、JSON 塞进 varchar、或其它「单字段承载多语义」的约定式拼接。
- 编辑界面中有独立语义的字段，应在数据模型中有**可查询、可校验、可约束长度**的对应存储（独立列、扩展表、或经批准的 JSON 列及 Schema），避免依赖人工解析文本。
- 历史已存在的拼接方案若要延续或迁移，须在对应模块 PRD 与实现方案中**单独立项**并写明兼容与迁移策略。
