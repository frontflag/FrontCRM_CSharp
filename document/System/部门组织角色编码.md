# 部门组织角色编码（固化标准）

与 `DataPermissionService` 组织层级、员工编辑页、`seed_dept_org_roles.sql` 保持一致。

| RoleCode | 含义 | 数据权限层级 |
|----------|------|----------------|
| `DEPT_DIRECTOR` | 部门总监 | 可看本部门路径下下属经理、员工的数据 |
| `DEPT_MANAGER` | 部门经理 | 可看本部门路径下下属员工的数据 |
| `DEPT_EMPLOYEE` | 部门员工 | 仅本人 |

## 初始化

在已有 `sys_permission` 的数据库上执行：

```text
seed_dept_org_roles.sql
```

脚本会：

- 插入/补齐上述三条角色（固定 `RoleId` 与 `seed_dept_demo_users.sql` 兼容）；
- 将历史 `DEPT_STAFF` 合并为 `DEPT_EMPLOYEE`（迁移 `sys_user_role` 后删除旧行）；
- 为总监/经理授予除 `rbac.manage` 外的全部 API 权限，为员工授予只读类权限集合。

## 历史兼容

- 库中仍可能存在的 `DEPT_STAFF`：由脚本统一为 `DEPT_EMPLOYEE`。
- 后端 `ResolveOrgRoleLevel` 仍识别 `DEPT_STAFF` 为员工级（1）。

## API

`POST/PUT .../rbac/admin/users` 中 `roleIds` 仅允许对应角色编码为：

`DEPT_DIRECTOR`、`DEPT_MANAGER`、`DEPT_EMPLOYEE`、`SYS_ADMIN`（后者用于维护系统管理员账号）。
