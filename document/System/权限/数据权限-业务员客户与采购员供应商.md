# 数据权限：业务员—客户 与 采购员—供应商

本文与 **`CRM.Core`** 中 **`DataPermissionService`** 当前实现一致，与 [业务规则总览](../业务规则总览.md) §2、[业务逻辑与数据关系文档](../业务逻辑与数据关系文档.md) §6.2 及 [RBAC 权限系统 PRD](../../PRD/RBAC权限系统PRD.md) 中数据权限描述同步维护。

---

## 一、共同前提

- 数据范围来自用户**主部门**（`RbacDepartment`）上的两个**独立**配置：  
  - **`SaleDataScope`**：销售数据范围  
  - **`PurchaseDataScope`**：采购数据范围  
- 取值约定：**`0=全部`，`1=自己`，`2=本部门`，`3=本部门及下级`，`4=禁止`**。  
- **`2` / `3`** 通过 **`GetAllowedUserIdsAsync`** 得到「当前用户 + 部门（及可选子部门）内可见用户 Id 集合」，再与业务数据上的**归属人**字段比对（含总监/经理看下属等组织规则）。  
- **系统管理员**（`IsSysAdmin`）在多数过滤方法中**不按数据范围缩小**。

---

## 二、业务员 ↔ 客户

| 项目 | 说明 |
|------|------|
| **绑定字段** | **`CustomerInfo.SalesUserId`**（归属业务员）。 |
| **范围配置** | **`SaleDataScope`**（与 `PurchaseDataScope` 无关）。 |
| **实现入口** | **`FilterCustomersAsync`**；销售订单列表为 **`FilterSalesOrdersAsync`**（**`SellOrder.SalesUserId`**），与客户侧逻辑同类。 |
| **规则摘要** | `PurchaseDataScope` 不用于客户列表；`SaleDataScope == 4` → 无客户；`== 1` → 仅 `SalesUserId == 当前用户`；`2`/`3` → `SalesUserId` 落在允许用户集合；`0`/管理员 → 不过滤。 |
| **财务例外** | **`FilterFinanceReceiptsAsync`**：部门 **`IdentityType == 5`（财务）** 时**不按客户业务员**缩小收款单范围。 |

**语义**：客户、销售订单等销售域列表「谁能看见」，由 **业务员归属字段 + 销售数据范围** 决定。

---

## 三、采购员 ↔ 供应商（当前实现）

### 3.1 供应商主数据（列表 / 详情 / `CanAccessVendorAsync`）

| 项目 | 说明 |
|------|------|
| **责任字段** | **`VendorInfo.PurchaseUserId`** 仍存在，用于主数据归属、统计、**付款单/进项发票**数据范围等。 |
| **列表范围** | **`FilterVendorsAsync`**：**仅当 `PurchaseDataScope == 4`（禁止）** 时返回空列表；**其余（0/1/2/3）及管理员**均返回**全部供应商**（便于报价等任选供应商）。**不按** `PurchaseUserId` 缩小可见集合。 |
| **专属供应商** | 后续在模型增加「专属」标识后，在 **`ApplyVendorExclusiveVisibilityFilter`**（或等价处）排除；当前未启用。 |

### 3.2 仍按采购数据范围 + 采购员维度的能力

| 对象 | 说明 |
|------|------|
| **采购订单** | **`FilterPurchaseOrdersAsync`**：按 **`PurchaseOrder.PurchaseUserId`** + **`PurchaseDataScope`**（与客户侧对称）。 |
| **付款单、进项发票** | **`FilterFinancePaymentsAsync` / `FilterFinancePurchaseInvoicesAsync`**：经 **`VendorId` → Vendor.PurchaseUserId`**，再按 **`PurchaseDataScope`** 与允许用户集合比对；**财务部（IdentityType=5）**不按供应商采购员缩小。 |
| **RFQ** | **`FilterRFQsAsync`** / 明细可见性：销售侧看 `SalesUserId`；采购侧看 **`AssignedPurchaserUserId1/2`** 与 **`PurchaseDataScope`**。 |

**语义**：**供应商档案谁都能看（除采购数据禁止）**；**谁经办哪张采购单、哪笔对供应商付款/收票**仍与 **采购员 / 供应商责任采购员 + 采购数据范围** 相关。

## 四、两者区别（对照）

| 对比项 | 业务员 — 客户 | 采购员 — 供应商（当前） |
|--------|----------------|-------------------------|
| **范围开关** | **`SaleDataScope`** | 供应商**列表**：主要看 **`PurchaseDataScope` 是否为 4**；采购单、对供应商财务单据用完整 **`PurchaseDataScope`**。 |
| **主数据归属字段** | **`SalesUserId`** | **`PurchaseUserId`**（列表**不再**用于缩小供应商集合）。 |
| **列表是否按归属人过滤** | **是**（1/2/3 下按业务员过滤）。 | **否**（供应商列表为全部，仅 **4** 为空）。 |
| **主要方法** | `FilterCustomersAsync`、`FilterSalesOrdersAsync` | `FilterVendorsAsync`（主数据）；`FilterPurchaseOrdersAsync`；`FilterFinancePaymentsAsync` 等。 |
| **财务** | 收款：`SalesUserId` + 销售范围；财务部门例外。 | 付款/进项：`Vendor.PurchaseUserId` + 采购范围；财务部门例外。 |

**一句话**：客户侧**始终**用「业务员 + 销售数据范围」管可见性；供应商主数据列表已**放开**以便报价选商，**采购执行与对供应商财务**仍按 **采购数据范围 + 采购员链路** 控制。
