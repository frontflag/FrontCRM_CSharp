# 到货通知、质检、入库业务流程文档

## 一、概述

### 1.1 业务背景

到货通知、质检、入库是供应链管理中紧密关联的三个核心环节，构成了商品从供应商到仓库的完整流程。

### 1.2 核心概念

| 概念 | 说明 |
|------|------|
| **到货通知** | 通知仓库准备收货的单据 |
| **质检** | 对到货商品进行质量检验 |
| **入库** | 将质检合格的商品入库到仓库 |

### 1.3 整体业务流程图

```
采购订单明细 → 创建到货通知 → 确认到货 → 创建质检单 → 质检结果 → 生成入库
                                              ↓
                                    ┌─────────┼─────────┐
                                    ▼         ▼         ▼
                                全部通过   部分通过   全部拒收
                                    ↓         ↓         ↓
                                全部入库   部分入库   生成异常单
                                          部分拒收
```

## 四、质检流程

### 4.1 创建质检单

**前置条件**:
1. 到货通知状态为"到货待检"
3. 同一到货通知尚未完成质检



### 4.2 质检结果处理

| 质检结果 | 入库状态 | 后续处理 |
|----------|----------|----------|
| 全部通过 | FulltockIn | 全部入库 |
| 部分通过 | PartStockIn | 部分入库，部分拒收 |
| 全部拒收 | Reject | 生成异常单 |

---

## 五、入库流程

### 5.1 质检生成入库

**触发条件**:
- 质检已完成
- 有通过的质检明细

**入库状态判定**:
```csharp
if (全部通过)      StockInStatus = FullStockIn;
else if (部分通过) StockInStatus = PartStockIn;
else if (全部拒收) StockInStatus = Reject;
```

### 5.2 入库后处理

1. 更新到货通知状态为"已入库"
2. 更新质检入库状态
3. 生成库存

## 六、核心业务规则

### 6.1 数量校验

| 校验项 | 规则 |
|--------|------|
| 质检数量 | ≤ 到货通知预期数量 |
| 入库数量 | ≤ 质检通过数量 |
| 明细数量 | 不能为0 |

### 6.2 状态流转关系

| 单据 | 状态流转 |
|------|----------|
| 到货通知 | 到货待 → 已质检 → 已入库 |
| 质检单 | 未入库→ 全部/部分入库 或 拒收 |
