# 1. 背景与目标

### 1.1 问题描述

当前系统缺乏统一的文档管理能力。

客户联系人照片、名片扫描件、销售合同、入库质检照片等文件分散存储，无法与具体业务记录关联，查找困难，容易丢失。

### 1.2 功能目标

- 支持多种格式文档上传：PDF、JPG、PNG、Word、Excel，zip
- 文档按业务场景分类存储在不同文件夹
- 文档与具体业务记录ID关联，支持一对多关系
- 提供文档列表查看和预览功能
- 规范的文件命名和存储结构

### 1.3 用户场景

以下为示例，包括但不限于以下场景。

场景一：上传销售合同
销售人员打开销售订单页面，点击上传文档按钮，选择合同文件，填写备注信息"Q1合同正式版"，点击上传。上传完成后，在该销售订单页面查看所有关联文档。点击文档可以预览文档。

场景二：上传采购合同
采购人员打开采购订单页面，点击上传文档按钮，选择合同文件，填写备注信息"Q1合同正式版"，点击上传。上传完成后，在该采购订单页面查看所有关联文档。点击文档可以预览文档。

场景三：上传供应商资料
采购人员打开供应商页面，可以上传供应商 工商执照，资质文档等。点击可放大预览。

场景三：上传客户名片
销售人员打开客户联系人详情页面，可以上传客户联系人名片扫描件、身份证照片等。点击名片图片可放大预览。

场景三：质检照片存档
仓库管理员办理入库时，拍摄质检照片，在入库单页面批量上传多张照片，系统按顺序保存，并记录上传时间和操作人，方便后续质量追溯。

# 2. 功能分解

### 2.1 上传文档

- 业务场景
    - 特定业务场景支持上传文档。包括：
        - 客户，客户联系人，供应商，供应商联系人，销售订单，采购订单，入库，库存，出库。
- 支持批量上传
    - 一次最多5个文档。
    - 单个文档不超过50M
- 上传文档时可以填写备注
    - 小于256个汉字
    - 非必须填写
- 上传检查
    - 文件格式
        - 过滤不支持格式。
            - 检查文件后缀。
        - 拦截伪装的exe恶意程序
            - 检查文件头格式。
    - 文件尺寸
        - 单个文档不超过50M

### 2.2 存储文档

- 文件存储在服务器特定文件夹下
    - 文件夹名称：UploadFile
- 分类存储
    - 不同业务场景在 UploadFile 文件夹下创建独立的子文件夹
    - 每条业务记录的文件，再创建字文件夹
        - 文件夹名称为业务记录编号
        - 相同业务编号的所有文档都存储在同一个业务记录编号的文件夹下
            - 包括缩略图
- 缩略图
    - 如果上传文档时图片格式，自动生成一份缩略图。
        - 图片文档存储会包含 原图+缩略图
- 文件命名
    - 存储时，文件会自动改名为存储文件名
    - 存储文件名命名规则是：
        - 第一部分是6位流水号，格式为两位年份、两位月份、两位日期加上6位当日顺序号，共12位数字
        第二部分是业务记录号，用于快速识别关联业务
        第三部分是原始文件名，保留用户上传时的文件名
            
            示例：250317000001_a1b2c3d4_采购合同.pdf
            
        - 确保并发情况下流水号不重复。

### 2.3 存储信息

- 记录业务字段和上传文档的数据关系
    - 便于数据查询
- 创建数据库表
    - 字段包括：
        - 业务场景，业务数据ID，原始文档名称，存储文件名称，文件字节大小，上传时间，上传人，上传备注，软删除标记，删除时间，删除人

### 2.3 查看文档

- 业务场景会有一个上传文档卡片页
    - 显示业务记录对应的上传文档名称，上传时间，上传人，上传备注

### 2.4 预览文档

- 点击上传，弹出预览窗口，显示文件内容
    - 支持格式：图片，pdf，word

### 2.5 管理文档

- 从系统管理角度，方便查找上传文档
    - 按条件查询上传文档，
        - 上传日期区间，上传人，备注内容，业务单号
- 可以删除文档
    - 软删除方式
- 可以下载文档

# 6. 配置要求

服务器存储路径配置
需在应用配置文件中指定文档存储的根目录路径，建议使用独立磁盘或独立分区，确保存储空间充足。

文件大小和数量限制配置
在配置文件中定义单文件大小限制（默认50MB）和单次上传文件数量限制（默认5个）。

允许的文件扩展名列表
在配置文件中定义系统支持的文件扩展名白名单。

# 7. 验收标准

功能验收

- 成功上传PDF、JPG、PNG、Word、Excel五种格式文件
- 上传后文件名格式符合规范：12位日期流水号_8位业务ID_原始文件名
- 文件物理存储在正确的目录结构下
- 数据库记录与物理文件一一对应，信息准确
- 备注信息成功保存，长度超过255字符时拒绝并提示
- 文档列表按上传时间倒序排列，最新上传显示在最前
- 图片文件点击可放大预览，支持缩放和关闭
- PDF文件可在浏览器内嵌预览，支持翻页
- Word和Excel文件提供下载功能
- 删除操作执行软删除，数据库记录保留，标记删除状态

性能验收

- 单文件50MB上传在10秒内完成（局域网环境）
- 同时上传5个文件，所有文件均成功，无相互干扰
- 文档列表查询响应时间小于500毫秒（100条记录以内）

安全验收

- 上传可执行文件（如EXE）被拒绝，提示不支持的格式
- 上传超过大小限制的文件被拒绝，提示大小超限
- 尝试访问其他业务的文档返回权限错误
- 删除操作需要登录用户身份，记录操作人

# 8. 风险提示

存储空间风险
文档文件持续积累可能占满磁盘空间，需规划存储容量监控和扩容方案，或制定历史文件归档策略。

文件名冲突风险
虽然流水号机制确保文件名唯一，但极端并发情况下需确保数据库函数的原子性。

大文件上传风险
超过50MB的文件上传可能因网络不稳定导致失败，建议大文件分片上传或提供断点续传（未来版本）。

文件类型伪装风险
用户可能通过修改扩展名上传恶意文件，后端需通过文件头信息校验真实类型，不仅校验扩展名。
