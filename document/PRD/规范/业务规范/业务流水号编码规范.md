# 业务流水号编码 — 项目业务规范

## 适用范围

凡通过 `ISerialNumberService` / `SerialNumberService` 生成的**业务单号**（客户编码、订单号、收付款单号等），均遵循本规范。配置存储在 PostgreSQL 表 `sys_serial_number`；**`ModuleCode` 为程序内唯一键，`Prefix` 为对外可见的业务标识，全局唯一**。

## 编号形态

```
完整单号 = Prefix + 数值段（固定 5 位）
```

- **Prefix（业务前缀）**：长度 **2～16**，仅允许 **大写英文字母 `A–Z`、数字 `0–9`、下划线 `_`**。存库与生成时按大写处理（`Trim` + `ToUpperInvariant`）。
- **数值段**：将递增整数 **从 0 起** 编码为 **固定 5 位** 的「32 进制」字符串（高位不足补 `0`），与 Prefix **直接拼接**，中间无分隔符。

示例：`SO` + `00000` → `SO00000`；`PAY_DEL` + `00001` → `PAY_DEL00001`。

## 32 进制字母表（实现约定）

与 `SerialNumberService` 中常量一致，共 32 个字符（**不含**易混淆的 **I、O、J、Q**）：

```
0123456789ABCDEFGHKLMNPRSTUVWXYZ
```

- 单号中**仅出现上述字符 + Prefix 允许的字符**；数值段只来自该字母表。
- 5 位该进制可表示范围：0～32^5−1（超出时服务抛错，需扩容规则时再评审）。

## 序号与并发

- 每个 `ModuleCode` 对应一行配置；**`CurrentSequence` 每次成功取号后 +1**。
- 取号在事务内对该行执行 **`SELECT … FOR UPDATE`**，保证并发下不重复。
- 表上 **`ModuleCode`、`Prefix` 均唯一**（索引 `IX_sys_serial_number_ModuleCode`、`IX_sys_serial_number_Prefix`）。
- **`ResetByYear` / `ResetByMonth`**：若某模块开启，按 UTC 时间到达新年/新月时将 `CurrentSequence` 归零（当前多数业务模块未启用，以库内配置为准）。

## 库表与列（摘要）

| 列 | 说明 |
| -- | ---- |
| ModuleCode | 模块代码，唯一，与代码常量一致 |
| ModuleName | 中文名称，展示用 |
| Prefix | 对外前缀，`varchar(16)`，全局唯一 |
| SequenceLength | 历史字段；当前生成逻辑固定使用 **5** 位数值段 |
| CurrentSequence | 当前序号；种子中客户/供应商可为 **-1**，首次生成后为 **0** 起编码 |

新建库或补表可参考根目录 `create_serial_number_table.sql`（须与当前种子 Prefix/`varchar(16)` 一致）。

## 变更说明（2026-04）

- **Prefix 列**由短前缀扩展为 **`varchar(16)`**，以支持 `PAY_DEL` 等。
- **产品命名对齐**：如销售订单 `SOR`→`SO`、采购 `PUR`→`PO`、出库 `SOUT`→`STO`、销项 `OUTI`→`INVO`、付款模块 `PAY`→`PAY_DEL`、财务付款 `FNP`→`PAY` 等，**详见上表**。
- **已生成的历史单号不变**；新规则仅作用于变更后新产生的编号。部署须与已包含新规则的 **API 程序**同步。
