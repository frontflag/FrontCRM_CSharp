# 销售订单 / 采购订单 — 增值税与结算币别

## 适用范围

- **销售订单明细**（`SellOrderItem` 及前端展示、统计、后续扩展的打印/报表）。
- **采购订单明细**（`PurchaseOrderItem` 及前端展示、统计、采购订单打印报表等）。

## 业务规则（强制）

1. **结算币别为人民币（RMB，编码 `1`）**  
   - 可按国内增值税口径做价税拆分或展示税率（系统内采购订单打印报表默认按 **13%**（小数 `0.13`）推算增值税，与模版一致）。  
   - 具体是否「含税价」仍以业务录入约定为准；报表侧在明细分录无单独税率字段时，对 RMB 行使用上述默认率推算页脚「不含税金额 / 增值税 / 价税合计」。

2. **结算币别不是人民币（非 `1`，如 USD、EUR、HKD、JPY、GBP 等）**  
   - **增值税额视为 0**（不对该明细计提或展示增值税金额）。  
   - **增值税率视为 0**（明细行税率、汇总行有效税率均为 0）。  
   - 行金额在报表中按 **无税价 = 价税合计** 处理（即不对该行做 `(1+税率)` 反拆）。

3. **主单币别与明细币别**  
   - 以**明细行上的 `currency`（结算币别编码）**为准；若历史数据明细未落库币别，则回退为**采购单/销售单主表 `currency`**。  
   - 一单内若混有 RMB 与外币明细，页脚「不含税金额 / 增值税」为各行推算结果之和；页脚展示的汇总税率取 **有效税率**（`增值税 ÷ 不含税金额`，无税时为 `0`）。

## 实现参考（代码）

- 前端统一方法：`CRM.Web/src/constants/currency.ts` 中的 `settlementVatRateDecimal`、常量 `SETTLEMENT_RMB_VAT_RATE_DECIMAL`。  
- 采购订单打印报表：`CRM.Web/src/views/RFQ/PurchaseOrderReportPage.vue`（按行币别计算税率与页脚汇总）。

## 与《结算币别下拉规范》的关系

币别编码与下拉选项以 [结算币别下拉规范](./结算币别下拉规范.md) 为准；本规范仅约束 **增值税 / 税率** 与币别的关系，不改变币别枚举本身。
