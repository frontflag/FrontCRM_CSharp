# 业务员与采购员下拉规范（业务数据范围）

## 1. 文档目的

统一系统中**选择业务员**、**选择采购员**两类控件的可选人员范围，保证：

- 列表筛选、表单选人只出现**对应业务部门**下的账号；
- **销售 / 采购一线账号**登录时，选人范围收敛为**本人及下属**（与组织树、职级规则一致），避免越权浏览同部门其他团队人员。

本文与接口实现一致：`CRM.API` 中 `AuthController` 的 `GET /api/v1/auth/sales-users-tree`、`GET /api/v1/auth/purchase-users-tree`。

---

## 2. 适用范围

- 前端所有**业务员**、**采购员**的**下拉 / 级联选人**（含列表筛选条件中的「按人员 ID 筛选」）。
- 不约束：仅用于**姓名/账号关键词模糊**的文本框（如需求明细、销售订单明细列表中的「业务员名称」输入框），该类不展示人员列表，由后端按关键词过滤即可。

---

## 3. 业务员下拉（销售）

### 3.1 业务规则

1. **可选人员必须属于销售相关部门**  
   判定与后端一致：`IdentityType == 1（销售）`，或部门名称包含「销售」/「sales」（不区分大小写）。

2. **当前用户是否挂在销售组织**  
   - 若用户**主部门**为销售部门，或其在**任一人事关系**中属于销售部门，则视为「在销售部门」，取一个锚定销售部门用于下属计算（主部门优先，否则取第一条销售部门关系）。

3. **可选范围**  
   - **系统管理员**：先取「所有存在部门关系的用户」再按销售树过滤，即**全部销售相关部门下**的启用用户（在树中出现的）。  
   - **在销售部门的非管理员**：仅 **本人 + 下属**（按部门路径子树 + 角色职级：`GetAllowedUserIdsAsync` 规则，与采购员树一致）。  
   - **不在销售部门的非管理员**：**所有挂在销售相关部门的用户**（与采购侧「非采购部门登录展示全部采购相关人员」对称）。

### 3.2 前端统一用法

| 场景 | 必须使用 |
|------|----------|
| 表单级联选人（部门树 + 人） | `SalesUserCascader`（内部请求 `getSalesUsersTree` → `/api/v1/auth/sales-users-tree`） |
| 列表筛选 `el-select`（扁平选项） | `authApi.getSalesUsersForSelect()`（同上接口，展平叶子节点） |

**禁止**：使用 `GET /api/v1/auth/users`（`authApi.getUsers`）填充客户「业务员」筛选或任何「选业务员」控件。

### 3.3 当前实现位置（核对用）

- `CRM.Web/src/components/SalesUserCascader.vue`
- `CRM.Web/src/api/auth.ts`：`getSalesUsersTree`、`getSalesUsersForSelect`、`flattenSalesUserTreeForSelect`
- 使用页面示例：`CustomerList.vue`、`CustomerSearchPanel.vue`、`CustomerEdit.vue`、`RFQCreate.vue`、`RFQEdit.vue`、`QuoteCreate.vue`、`QuoteEdit.vue`、`SalesOrderCreate.vue`、`SalesOrderDetail.vue`（编辑业务员）

---

## 4. 采购员下拉（采购）

### 4.1 业务规则

1. **可选人员必须属于采购相关部门**  
   `IdentityType` 为采购相关枚举值，或部门名包含「采购」/「purchase」（不区分大小写）。

2. **锚定采购部门**  
   主部门或人事关系中是否在采购部门（`ResolvePurchaseAnchorDepartmentId`）。

3. **可选范围**  
   - **系统管理员**：全部挂部门用户后按采购树过滤。  
   - **在采购部门**：**本人 + 下属**（`GetAllowedUserIdsAsync` + 锚定采购部门）。  
   - **不在采购部门**：**全部采购相关部门下的用户**。

### 4.2 前端统一用法

| 场景 | 必须使用 |
|------|----------|
| 表单级联选人 | `PurchaserCascader`（`getPurchaseUsersTree` → `/api/v1/auth/purchase-users-tree`） |
| 列表筛选 `el-select` | `authApi.getPurchaseUsersForSelect()` |

**禁止**：使用 `authApi.getUsers` 填充供应商「采购员」筛选或任何「选采购员」控件。

### 4.3 当前实现位置（核对用）

- `CRM.Web/src/components/PurchaserCascader.vue`
- `CRM.Web/src/api/auth.ts`：`getPurchaseUsersTree`、`getPurchaseUsersForSelect`、`flattenOrgUserTreeLeaves`
- 使用页面示例：`VendorList.vue`、`VendorSearchPanel.vue`、`VendorEdit.vue`、`PurchaseOrderCreate.vue`、`QuoteCreate.vue`、`QuoteEdit.vue`、`RFQDetail.vue`（分配采购员）、`QcCreate.vue`

---

## 5. 后端接口摘要

| 接口 | 说明 |
|------|------|
| `GET /api/v1/auth/sales-users-tree` | 销售组织树；人员节点可带 `userName`、`realName` 供前端展示 |
| `GET /api/v1/auth/purchase-users-tree` | 采购组织树 |
| `GET /api/v1/auth/users` | 全站启用用户列表，**不应用于**本规范约束的选人场景 |

实现类：`CRM.API/Controllers/AuthController.cs`（`GetSalesUsersTree`、`GetPurchaseUsersTree`、`ResolveSalesAnchorDepartmentId`、`ResolvePurchaseAnchorDepartmentId`、`GetAllowedUserIdsAsync`）。
