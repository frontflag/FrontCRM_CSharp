# 省市地区级联选择控件规范

## 1. 定义与适用范围

**省市地区级联选择控件**指表单中用于选择**行政区划（省 / 市 / 区）**的 `el-cascader` 封装：支持**收藏快捷入口**、**省/市两级即可确认**（区可选）、**本地持久化收藏列表**，并与深色主题表单项样式一致。

| 项目 | 说明 |
|------|------|
| **实现组件** | `CRM.Web/src/components/RegionCascaderWithQuickPick.vue` |
| **数据源** | `CRM.Web/src/data/regions.ts`（静态省市区树；含香港、台湾等扩展项） |
| **地区占位与回显** | `CRM.Web/src/constants/region.ts`（`REGION_DISTRICT_PLACEHOLDER`、`regionCascaderValueFromFields` 等） |
| **收藏持久化** | `CRM.Web/src/composables/useFavoriteRegions.ts`（`localStorage` 键 `crm_region_favorites_v1`） |

**当前接入页面（示例）**

- 客户编辑：`CustomerEdit.vue`（字段「地区」）
- 客户地址弹窗：`AddressDialog.vue`（「所在地区」）

**不在本规范内**：非级联的纯文本地区展示、后端 `Province/City/District` 字段长度与校验策略（以接口与模型为准）。

---

## 2. 交互与业务逻辑

### 2.1 可选层级（省 / 市 / 区）

- 用户**至少选到省 + 市**即可作为有效地区；**区可选**。
- 仅选到**市**（在「市」这一级确认）时，视为**未选区**：表单中 **`district` 写入占位符** `—`（常量 `REGION_DISTRICT_PLACEHOLDER`，Unicode U+2014）。
- 选到**区**时，`district` 为真实区县名称。
- **仅选省（单段路径）**：若级联在严格模式下产生单段路径，表单可只带省、清空市/区；业务上建议以「保存校验」要求至少到市（实现可选）。

### 2.2 输入框展示

- 使用分隔符 **`/`**（与 `el-cascader` 的 `separator` 一致）。
- 两级：`省/市`；三级：`省/市/区`。

### 2.3 「收藏」入口（第一列首项）

- 第一列**第一项**为 **「收藏」**，用于展开右侧子菜单，列出已保存的收藏项。
- **「收藏」本身不得作为省/市值**：选项上对该节点设置 **`disabled: true`**，在 `checkStrictly` 下仍可**点击展开子级**，但**不会**被选为最终值写入表单（避免出现输入框显示「收藏」）。
- 子菜单中每一项对应一条收藏路径（见下）；无数据时可为灰色 **「暂无收藏」**（不可选）。

### 2.4 收藏（星标）

- 在**省级**、**市级**两行左侧显示**五角星**（空心 / 实心表示未收藏 / 已收藏）。
- **点击星标**：切换该逻辑路径的收藏状态（`@click.stop`，不触发行选中）。
- **收藏路径规则（与当前实现一致）**
  - **省行**：`[省, 该省下第一个市]`（两段）。
  - **市行**：`[省, 市]`（两段）。
- 「收藏」分支下的行**不显示**星标；收藏子项右侧另有「取消收藏」按钮（见下）。

### 2.5 收藏子菜单：取消收藏

- 每条收藏文案右侧提供 **关闭（×）** 按钮，**仅移除本地收藏**，不将该项作为地区选中（`@click.stop`）。
- 移除后列表立即更新（`localStorage` 同步）。

### 2.6 从「收藏」选中某条

- 用户点击收藏子项中的**具体地区**后，将解析出的 **`[省, 市]` 或 `[省, 市, 区]`** 写回表单；不得把内部合成值（如 `__crm_favorites__`）当作省名存储。

### 2.7 打开下拉时的定位

- 依赖 Element Plus：下拉展开时会对齐 / 滚动到当前选中路径（内部 `scrollToExpandingNode`）。**控件侧 `modelValue` 须与 `regions` 树一致**（两段或三段真实地名），以便正确定位。

### 2.8 表单与回显约定

- **变更回调**：`length >= 2` 时写入 `province`、`city`；`district` 为第三段或占位符 `—`；清空时同步清空地区相关字段。
- **回显**：用 `regionCascaderValueFromFields(province, city, district)`：若区为占位或空，级联绑定 **`[省, 市]`**；否则 **`[省, 市, 区]`**。
- **详情/地址展示**：拼接地址时应对占位区做过滤（例如详情页 `formatFullAddress` 不把 `—` 拼入展示串）。

## 3. 技术实现要点（Element Plus）

封装内对 `el-cascader` 使用统一 **`props`**（与实现保持同步）：

| 属性 | 值 | 说明 |
|------|-----|------|
| `expandTrigger` | `click` | 点击展开 |
| `checkStrictly` | `true` | 可选任意一级（省/市可单独成为路径节点） |
| `emitPath` | `true` | 值为路径数组 |
| `checkOnClickNode` | `true` | 点击节点即可选中（配合严格模式） |
| `showPrefix` | `false` | 隐藏默认单选圆点前缀，避免与星标抢视觉 |

- 自定义 **`#default`（render-label）** 渲染：星标 + 文案 +（收藏子项右侧）取消按钮。
- 合成选项：根节点「收藏」`value` 为内部常量（如 `__crm_favorites__`）；收藏子项 `value` 以 `__fav::` 前缀编码，**仅在组件内解析**，不得提交到业务接口。

## 4. UI 与样式约定

- **星标按钮**：小号方形可点区域，空心 / 实心星；悬停时略提亮；已收藏为强调色（与主题变量一致）。
- **取消收藏按钮**：小号图标按钮，默认次要色，悬停偏**危险色**提示删除含义。
- **行内布局**：左侧星标（若有）+ 中间文案（省略号）+ 右侧取消（若有）；`min-width: 0` 避免撑破列宽。
- **下拉面板**：可通过 `popper-class`（如 `region-cascader-popper`）统一内边距；与表单控件同用时，客户编辑页可传 **`cascader-class="q-cascader"`** 以匹配暗色 `el-input` 皮肤。

---

## 5. 数据与持久化

| 项目 | 说明 |
|------|------|
| 收藏条数上限 | 默认最多 **12** 条（实现见 `useFavoriteRegions`） |
| 存储位置 | 浏览器 `localStorage`，键名 **`crm_region_favorites_v1`** |
| 收藏项结构 | `path: string[]`（2 或 3 段）+ 展示用 `label`（一般为 `path.join(' / ')`） |
| 多页面 | 同一站点共用同一键，客户编辑与地址弹窗**共享**收藏列表 |

---

## 6. 维护说明

- 调整省市区数据源时，只改 `regions.ts`（及必要的种子/文档），并验证收藏路径仍能在树中定位。
- 修改占位符、级联 `props` 或「收藏」合成规则时，须同步更新**本规范**与 `constants/region.ts` 注释。
- 新增使用本控件的页面：传入 `v-model`（`string[]`）、`@change` 与 `CustomerEdit` / `AddressDialog` 的表单字段约定保持一致。
