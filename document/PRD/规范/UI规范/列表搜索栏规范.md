# 列表搜索栏规范

## 1. 定义与适用范围

**列表搜索栏**指数据列表页中、位于**页面标题区下方**、**数据表格上方**的一行（可多行折行）**筛选与检索控件带**：用于组合关键词、枚举筛选、日期区间与「搜索 / 重置」操作。

- **本规范来源**：`/customerlist`（`CustomerList.vue`）中 `.search-bar` 及其子元素的**布局与样式**（不含业务字段含义与各 `el-option` 数据）。
- **适用场景**：各类列表页（客户、供应商、订单、采购申请等）需与当前主站深色主题**视觉与间距一致**时，应优先复用本结构的 **类名与 SCSS 规则**；具体筛选项数量与顺序可按业务裁剪。

**不在本规范内**：列表页标题栏（`.page-header`）、表格与分页本体；全局 `crm-unified-list.scss` 中的表格皮肤。

---

## 2. 结构约定（DOM）

### 2.1 容器层级

```
.search-bar
  └── .search-left
        ├── .filter-field-label（可选，每个独立控件块前可重复）
        ├── .search-input-wrap
        │     ├── svg.search-icon（放大镜）
        │     └── input.search-input
        ├── el-select.status-select（可多个；可加修饰类如 .status-select--sales）
        ├── el-date-picker.filter-date-range（可选）
        ├── button.btn-primary.btn-sm「搜索」
        └── button.btn-ghost.btn-sm「重置」
```

### 2.2 布局类职责

| 类名 | 职责 |
|------|------|
| `.search-bar` | 外层横向条：与表格之间留间距，右侧可预留扩展（当前实现多为单 `.search-left` 占满） |
| `.search-left` | **主的内容行**：`flex` + `flex-wrap`，控件横向排列，窄屏自动折行 |
| `.filter-field-label` | 紧挨输入/下拉的**短标签**（如「关键词」），12px、muted 色、不换行 |
| `.search-input-wrap` | 包裹**原生** `input` + 左侧内嵌搜索图标 |
| `.search-input` | 单行关键词输入，默认宽度见 §4；可用 BEM 修饰类（如 `--narrow`）在业务页收窄 |
| `.status-select` | `el-select` 深色皮肤与统一宽度；业务员等较长文案可加 `.status-select--sales` 加宽 |
| `.filter-date-range` | `el-date-picker` `type="daterange"` 的固定宽度与内部输入框压缩规则 |
| `.btn-primary` / `.btn-ghost` | 搜索 / 重置；使用 **`.btn-sm`** 与列表页尺度匹配 |

## 3. 布局与间距（CSS）

以下为与客户列表**一致**的规范值（实现时引用 `@/assets/styles/variables.scss` 中的 token）。

| 规则 | 值 |
|------|-----|
| `.search-bar` | `display: flex; align-items: center; justify-content: space-between; margin-bottom: 12px;` |
| `.search-left` | `display: flex; align-items: center; gap: 10px; flex-wrap: wrap;` |
| `.filter-field-label` | `font-size: 12px; font-weight: 500; color: $text-muted; white-space: nowrap;` |
| `.search-input-wrap` | `position: relative; display: flex; align-items: center;` |
| `.search-icon` | `position: absolute; left: 10px; color: $text-muted; pointer-events: none;` |
| `.search-input` | 默认 `width: 220px`; `padding: 7px 12px 7px 32px`; `background: $layer-2`; `border: 1px solid $border-panel`; `border-radius: $border-radius-md`; `color: $text-primary`; `font-size: 13px`; `font-family: 'Noto Sans SC', sans-serif`; `outline: none`; `transition: border-color 0.2s`；`::placeholder` 使用 `$text-muted`；`:focus` 边框 `rgba(0,212,255,0.4)` |
| `.status-select` | 默认 `width: 120px`；`:deep(.el-select__wrapper)` 无 box-shadow，背景 `$layer-2`，边框 `$border-panel`，圆角 `$border-radius-md`；占位符与选中项颜色与主题一致 |
| `.status-select--sales` | `width: 150px`（业务员等较长 label） |
| `.filter-date-range` | `width: 218px`; `flex-shrink: 0`；内部 `el-range-editor` 与 `el-range-input` 字号、宽度与客户列表一致（双日期 +「至」不换行混乱） |

**页面上下文**（推荐与列表页统一）：

- 页面容器：`padding: 24px`; `min-height: 100%`; `background: $layer-1`; `font-family: 'Noto Sans SC', sans-serif`。
- 字体：可按需引入 `Noto Sans SC`（与 `CustomerList.vue` 一致）。

## 4. 操作按钮

| 按钮 | 类 | 规范 |
|------|-----|------|
| 搜索 | `btn-primary btn-sm` | 青蓝渐变、白字、小尺寸（`padding: 6px 12px`; `font-size: 12px`）；`hover` 轻微上移 + 外发光（与客户列表一致） |
| 重置 | `btn-ghost btn-sm` | 透明底、边框 `$border-panel`、muted 字色；`hover` 边框与字色略提亮 |

加载中可对按钮加 `disabled` 并降低透明度（业务页自选）。

---

## 5. Element Plus 与交互约定

| 项目 | 约定 |
|------|------|
| `el-select` | 列表页检索条内建议使用 **`:teleported="false"`**，避免下拉层级与侧栏/遮罩异常；`clearable` 按业务开启 |
| 筛选变更 | **下拉变更即搜**（`@change="handleSearch"`）可减少「点了筛选却忘点搜索」的落差；若后端压力大可改为仅按钮触发，但须统一产品行为 |
| 关键词 | `input` 使用 **`@keyup.enter="handleSearch"`** |
| 日期区间 | `@change` 内在业务侧写入查询参数并触发列表刷新（与客户列表 `onCreatedRangeChange` 模式一致） |
| `el-date-picker` | `value-format="YYYY-MM-DD"`、`range-separator="至"`、占位「创建起 / 创建止」等为参考文案，可按业务改名 |

## 6. 字段顺序（参考，非强制）

客户列表当前顺序为：

**关键词 → 状态 → 级别 → 类型 → 行业 → 业务员 → 创建日期区间 → 搜索 → 重置**

其他列表建议遵循：

1. **最广的文本检索**（关键词 / 单号）靠前；  
2. **高频枚举**（状态）次之；  
3. **辅助维度**（类型、人员、日期）随后；  
4. **搜索 / 重置**始终在同一行末尾（折行时尽量保持二者相邻）。

---

## 7. 扩展与裁剪

- **多个关键词框**：每个块前加独立 `.filter-field-label`，多个 `.search-input-wrap` + `.search-input`；可通过修饰类收窄宽度（如 `.search-input--narrow { width: 180px; }`）。
- **无「标签 + 输入」**：可省略 `.filter-field-label`，仅保留 `.search-input-wrap`（与销售订单列表等页一致）。
- **筛选项少**：仍建议使用相同 `.search-bar` / `.search-left` / 按钮样式，保持全站一致。
- **禁止**：在检索条内使用浅色未覆盖的 `el-input` 默认白底作为唯一主文本框（破坏深色规范）；应使用本规范的 **原生 `input.search-input`** 或同步做深度主题覆盖。
