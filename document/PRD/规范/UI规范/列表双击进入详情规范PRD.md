# 列表双击进入详情规范 PRD（V1.0）

## 1. 文档目的

统一「业务列表 → 详情 / 主记录页」的交互：**凡列表行对应有独立详情页（或系统内约定的主查看页）时，用户双击数据行应能跳转**，与单击选中、高亮、展开侧栏等行为区分开，降低误触。

---

## 2. 适用范围

- 前端所有**主数据 / 单据列表**（`el-table` / `CrmDataTable`）
- **卡片式列表**（非表格）中，整块记录对应详情页时，同样适用（双击卡片主体区域）

不包含：

- 详情页内嵌的子表、弹窗内表格、仅用于编辑行内数据的表格
- **无**对应详情或主查看路由的列表（例如仅「标记完成」「执行出库」等流程动作、且无统一详情 URL 的页面——在实现中可不绑双击跳转，并在评审时注明原因）

---

## 3. 交互规则（强制）

1. **双击行（或卡片主体）**：跳转至该行对应的详情页；若业务上仅有「编辑页」作为主入口（如部分系统管理列表），则双击进入**编辑页**，与现有「编辑」按钮行为一致。
2. **单击行**：不因此强制跳转详情；仍用于当前页已有语义（如 `highlight-current-row`、选中、打开侧栏、勾选等）。
3. **操作列**：在已绑定「行双击进详情」的表格中，操作列单元格内应 **`@dblclick.stop`**（或等价阻止冒泡），避免用户快速双击按钮时触发行级双击跳转。
4. **与链接/按钮并存**：列内「单号链接」「详情」等仍可保留**单击**打开；双击空白数据区同样进详情，行为一致即可。
5. **实现方式**：
   - 使用 `CrmDataTable` 时：在表组件上监听 **`@row-dblclick="..."`**（属性会透传至内部 `el-table`）。
   - 原生 `el-table`：使用 **`@row-dblclick`**。

---

## 4. 与相关规范的关系

- 与《列表操作按钮颜色规范》并行：操作列颜色按该规范；本规范只约束**进入详情的触发方式（双击行）**及操作列双击冒泡处理。
