# 列表操作按钮颜色规范 PRD（V1.1）

## 1. 文档目的

统一系统中列表页“操作列”的按钮颜色语义，确保：

- 用户一眼区分安全操作与高风险操作
- 不同页面的交互认知一致
- 新增页面有可执行标准，避免随意配色

---

## 2. 适用范围

- 前端所有列表页（`el-table` / `CrmDataTable`）
- 操作列（通常为“操作”列）中的行级动作按钮
- 包含但不限于：客户、供应商、RFQ、订单、物流、库存、财务列表

---

## 3. 颜色语义总规则（强制）

1. **数据查看/编辑类动作固定蓝色（`primary`）**
   - 例如：`查看`、`详情`、`编辑`、`预览`、`下载`

2. **高危动作固定红色（`danger`）**
   - 例如：`删除`、`作废`、`驳回`、`解除绑定`、`强制关闭`

3. **「新建 / 新增 / 创建」类动作固定绿色（`success`）**
   - 语义：在系统中**新增一条业务数据或业务对象**（从无到有），而非在既有流程上推进状态
   - 例如：`新建`、`新增`、`创建`、`创建订单`（入口型）、列表工具栏「新建客户」等
   - 注意：若文案为「创建」但实为流程节点动作（如流程内「创建子任务」且与流转强绑定），按第 4 条归入业务流转，**不使用**绿色

4. **业务流转动作禁止使用蓝色 / 红色 / 绿色**
   - 必须使用 `warning` / `info` 等**非** `primary` / `danger` / `success` 语义色（含 Element Plus 默认次要按钮等中性样式亦可）
   - 例如：`提交审核`、`生成报价`、`生成采购单`、`确认到货`、`申请出库`、`审批通过`

5. **同一行多个业务动作应尽量使用不同颜色**
   - 目标：用户无需阅读文字也可初步区分动作类型

---

## 4. 推荐颜色映射（默认）

> 未在下表列出的业务动作，按“流程推进/中间处理/辅助查看”就近归类。

- `primary`（蓝）：
  - 查看类：查看、详情、编辑、预览、下载
- `danger`（红）：
  - 高危类：删除、作废、驳回、解除、取消（不可逆或高影响）
- `success`（绿）：
  - **仅**新建/新增/创建类：新建、新增、创建（见 §3 第 3 条与边界说明）
- `warning`（黄）：
  - 业务流转类（流程推进、待确认、提交、生成下游单据等）：提交审核、生成报价、生成采购单、申请采购、审批通过、确认到货、申请出库、提交申请
- `info`（灰/青）：
  - 辅助操作：明细、记录、日志、只读辅助入口；或与 `warning` 并存时的次要流转动作

---

## 5. 实施要求

1. 新增列表页必须遵循本规范，不得将业务流转动作设为 `primary` / `danger` / `success`。
2. 旧页面改造时，优先修正“业务动作误用蓝/红/绿”的按钮；并核对「新建/新增/创建」是否统一为绿色。
3. 同一页面至少保证：
   - 查看/编辑类统一蓝色
   - 删除/作废类统一红色
   - 新建/新增/创建类统一绿色
   - 业务流转类统一为黄/灰（`warning` / `info` 等），且不与绿混用
4. 若存在权限控制导致按钮显隐变化，不影响颜色语义规则。

---

## 6. 验收标准

1. 任一列表页操作列中：
   - 查看/编辑为蓝色
   - 删除/作废为红色
   - 新建/新增/创建为绿色
   - 业务流转动作不出现蓝色、红色或绿色
2. 至少 2 个业务动作并存时，颜色应可区分（不全部同色）。
3. 抽查页面（RFQ/订单/物流/财务）满足上述规则。
