# 扩展面板.帮助规范

本文约定工作台**右侧「帮助」页签**（扩展面板.帮助）所展示的 Markdown 帮助手册的**编写要求**与**前端渲染样式**，与各业务列表/详情页的交互说明保持一致。

实现入口：`CRM.Web/src/components/workspace/HelpManualPanel.vue`（类名 `help-manual-panel` / `help-md`）；文档资源位于仓库根目录 `help/`，构建时同步至 `CRM.Web/public/help/`。

---

## 1. 文档资产与路由映射

| 项 | 约定 |
|----|------|
| 注册表 | `help/menu-registry.json`：为每个主菜单项配置 `id`、`label`、关联的 `routeNames` 与 `pathPrefixes`。 |
| 单页路径 | `help/pages/{菜单名称}_{菜单ID}.md`，例如 `pages/销售订单_MENU_SALES_ORDERS.md`。名称需与注册表 `label`、`id` 一致，便于脚本与人工检索。 |
| 总目录 | `help/帮助文档目录.md` 由同步脚本维护，列出各菜单页链接。 |
| 同步 | 开发/构建前执行 `scripts/sync-help.mjs`（通常挂在 `npm run dev` / `build`）：将 `help/` 复制到 `CRM.Web/public/help/`，**不覆盖已有页面文件**；运行态通过 `fetch` 加载 `public/help` 下内容。 |
| 路由解析 | `CRM.Web/src/utils/helpDocPath.ts`：按当前路由匹配注册表项，解析默认打开的 `.md` 相对路径；面板内点击指向其它 `.md` 的链接时**仅在面板内切换**，不整页跳转。 |

---

## 2. 单页 Markdown 内容结构

### 2.1 推荐顺序

1. **返回目录**：首行使用 Markdown 链接回到总目录，例如  
   `[帮助文档目录](../帮助文档目录.md)`  
2. **页面主标题**：一级标题使用 `# {菜单名称}`，或与下文「标题微调」一致使用 HTML。  
3. **`## 页面功能`**：用 bullets 简述列表/筛选/权限敏感字段等（权限码用反引号包裹）。  
4. **`## 操作说明`**：按 **2.2** 的版式编写；**不要**再使用「操作 | 说明 | 前置条件」三列表格作为默认形态（遗留页可逐步迁移）。

### 2.2 操作说明版式（卡片块）

操作说明应以**一块操作 = 一列三行语义**呈现，便于扫读：

- **第一行**：操作名称，全文加粗（独立一段）。  
- **第二行**：`**说明：**` 起头，描述行为结果或业务含义。  
- **第三行**：`**前置条件：**` 起头，描述状态、权限、是否需确认等。

技术写法（与 `marked` GFM 兼容）：每个操作包在带 class 的 HTML 容器内，容器内需**空行**以便 Markdown 被解析为段落。

```markdown
<div class="help-op-block">

**提交审核**

**说明：** 主订单状态由「新建」流转为「待审核」。

**前置条件：** 状态为「新建」且账号具备 `sales-order.write` 权限。

</div>
```

样式由扩展面板.帮助统一提供（见第 3 节 `.help-op-block`）。

### 2.3 状态与枚举表述

- **说明**、**前置条件**中出现的订单/流程状态须写**业务状态名称**（如「新建」「待审核」），避免仅写 `1→2`、`status 1->2` 等裸枚举值。  
- 枚举与产品常量对齐（例如销售订单主状态与前端 `SALES_ORDER_STATUS_TEXT` / 后端主状态枚举一致）。

### 2.4 不必写入「操作说明」的操作

以下属于通用交互，用户无需在帮助中重复：**详情**、**编辑**、**删除** 等可在「操作说明」中**整卡省略**；仅当某页对_delete_ 有特殊入口（例如仅详情页「更多」中提供）且易误解时，再视需要单独说明（当前销售订单帮助已统一省略删除卡）。

有**非通用**或**易误解**的操作（提交审核、生成采购单、业务特有按钮等）应保留卡片并写清说明与前置条件。

### 2.5 主标题位置微调（可选）

若需在「帮助文档目录」链接与主标题之间增加垂直间距，可将一级标题写为 HTML 并加 class：

```html
<h1 class="help-h1--offset-down">销售订单</h1>
```

效果：在通用 `h1` 样式基础上另加 `margin-top: 8px`。其它页面默认仍使用 `# 标题` 即可。

## 3. 前端展示样式（`help-md`）

以下为扩展面板.帮助对 Markdown 渲染结果的约定样式，修改须兼顾**可读性**与**暗色主题**一致性。

| 选择器 / 区域 | 约定 |
|---------------|------|
| 面板基础 | 正文字号约 12px，行高约 1.55，默认字色偏亮灰蓝。 |
| `h1` | 19px，字重 600，白色，下边框 `rgba(0,212,255,0.15)`。 |
| `h1.help-h1--offset-down` | 在 `h1` 基础上 `margin-top: 8px`。 |
| `h2`（如「页面功能」「操作说明」） | 16px，字重 **400**（不加粗），白色。 |
| `h3` | 14px，字重 600，白色。 |
| 段落 / 列表 | 适度外边距；列表缩进统一。 |
| `code` | 等宽字、小字号、浅青底与字色，块级与行内区分 `pre` 与行内 code。 |
| 链接 `a` | 亮色下划线，hover 可依赖全局；点击行为见 `resolveHelpLinkHref`。 |
| `table` | 仍支持旧文档表格：细边框、表头浅底；新文档优先卡片块。 |
| `.help-op-block` | 卡片：圆角、浅青描边、弱青底；内部段落间距收紧；**首段内 `strong`（操作名）** 略大、字重 700，与「说明」「前置条件」区分层级。 |

样式实现集中维护于 `HelpManualPanel.vue` 的 scoped `:deep(.help-md …)`，新增语义时应**先补样式再写文档**，避免 class 无效。

## 4. 参考样例

- **页面文件**：`help/pages/销售订单_MENU_SALES_ORDERS.md`（同步副本：`CRM.Web/public/help/pages/…`）。  
- **特点**：目录链接 + `help-h1--offset-down`；「页面功能」简明；「操作说明」仅为**提交审核**、**生成采购单**两卡；状态用语义名称；无详情/编辑/删除占位卡。

---

## 5. 新增菜单时的检查清单

1. 在 `menu-registry.json` 增加条目并确认 `routeNames` / `pathPrefixes` 能命中实际路由。  
2. 在 `help/pages/` 添加 `pages/{label}_{id}.md`（可先占位再迭代）。  
3. 执行同步脚本，确认 `CRM.Web/public/help/` 下存在对应文件。  
4. 在业务路由下打开右栏「帮助」，确认默认打开正确文档、目录链接与站内 `.md` 跳转正常。  
5. 「操作说明」优先使用 **`.help-op-block` 三行结构**；枚举写名称；省略通用操作卡。
