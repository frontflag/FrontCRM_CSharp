# 客户 / 供应商名称状态展示规范 PRD

## 目标

在列表、详情、黑名单等场景中，统一**客户名称**与**供应商名称**在「冻结」「黑名单」状态下的视觉表达，便于一眼识别风险主体，并与业务字段（客户 `DisenableStatus` / `BlackList`，供应商 `IsDisenable` / `BlackList`）一致。

## 适用范围

- 客户：客户列表、客户详情页标题与基本信息中的主名称、客户黑名单列表中的名称展示。
- 供应商：供应商列表、供应商详情页标题与基本信息中的主名称、供应商黑名单列表中的名称展示。
- 其他页面若展示客户/供应商主名称且具备上述状态字段，应遵循本规范。

## 规则

### 1. 名称颜色（灰色）

当主体满足以下**任一**条件时，**主名称**文字使用弱化灰色（与详情页标题 `page-title--muted` 一致：`rgba(150, 170, 195, 0.82)`）：

- 冻结 / 禁用：`true`
- 黑名单：`true`

即：**黑名单或冻结（禁用）时，名称显示为灰色。**

### 2. 冻结：名称后雪花图标

当 **冻结** 为 `true` 时，在**主名称**（列表为名称列主行、详情为页标题名称）**之后**紧跟展示**冰晶雪花**图标（渐变冰蓝色，实现见 `PartyStatusIcons` 组件）。

- 客户字段：`disenableStatus === true`
- 供应商字段：`isDisenable === true`

### 3. 黑名单：名称后骷髅图标

当 **黑名单** 为 `true` 时，在**主名称**之后展示**骷髅 + 交叉骨**图标（当前产品约定为白色填充 + 轻微暗色描边式阴影，实现见 `PartyStatusIcons`）。

- 客户 / 供应商字段：`blackList === true`

### 4. 同时冻结且黑名单

两种图标**并列**展示于名称之后，顺序为：**雪花 → 骷髅**（与状态无优先级冲突，仅视觉顺序约定）。

### 5. 尺寸

- **详情页标题旁**：`PartyStatusIcons` `size="md"`（图标约 22px）。
- **列表 / 黑名单卡片名称旁**：`size="sm"`（图标约 16px）。

### 6. 详情页基本信息区

除标题外，**基本信息**中「客户名称 / 供应商名称」字段值在处于冻结或黑名单时，使用相同灰色（`info-value--party-muted`），**不再重复**标题旁已展示的图标，避免冗余。

## 前端实现约定

- 统一使用组件：`CRM.Web/src/components/party/PartyStatusIcons.vue`。
- 传入 `entityId`（行或详情主键）以保证 SVG 渐变 / mask 的 `id` 在列表中唯一。
- 文案提示：`title` / `aria-label` 分别为「冻结」「黑名单」。

## 后端字段对照

| 业务     | 冻结 / 禁用     | 黑名单    |
|----------|-----------------|-----------|
| 客户     | `DisenableStatus`（前端 `disenableStatus`） | `BlackList`（`blackList`） |
| 供应商   | `IsDisenable`（前端 `isDisenable`）         | `BlackList`（`blackList`） |

列表与详情接口需返回上述布尔字段，前端方可完整展示。
