# 业务列表规范

本规范沉淀所有「业务列表页」的通用 UI 与交互约定，作为新页面开发与代码评审的统一基线。

## 1. 列表表头（列设置：显示/隐藏/顺序/持久化）

### 1.1 目标

- **用户可自定义列顺序**（拖拽）
- **用户可自定义列显示/隐藏**（勾选）
- **设置需持久化**，下次进入仍生效
- **可一键恢复默认**

### 1.2 入口位置（重要）

- **入口不占用列表上方额外一行**（避免浪费纵向空间）。
- 入口放在**列表底栏左侧**（与分页同一行），并与右侧分页控件**顶部对齐**。
- 入口按钮仅显示**图标**（齿轮），不显示“列设置”文字；“列设置”通过 **Tip/Tooltip** 展示。

> 参考实现：`CRM.Web/src/views/RFQ/RFQItemList.vue`（需求明细）

### 1.3 底栏布局规则（间隔区）

- “列设置”按钮右侧必须有一个**空白间隔区域**（Spacer），其他按钮从间隔区右侧开始摆放。
- “复选篮子”入口与相关按钮按同一逻辑放置（从 Spacer 右侧开始）。

> 参考实现：`RFQItemList.vue` 的 `.table-footer-bar` / `.table-footer-left` / `.table-footer-spacer`

### 1.4 列设置交互（抽屉）

- 点击齿轮打开右侧抽屉「列显示与顺序」。
- 抽屉内：
  - 勾选控制显示/隐藏
  - 拖拽控制顺序（仅对“中间列”生效）
  - “恢复默认”一键重置为页面初始列状态
- 文案提示需说明：设置保存在本机（localStorage），下次打开仍有效。

### 1.5 固定列与不可配置列

- **勾选列**（`type=selection`）必须固定在最前（pinned start），且 **不可隐藏**。
- **操作列**（右侧固定列，通常 `fixed='right'` 且 `class-name='op-col'`）必须固定在最后（pinned end），且 **不可隐藏**、**不可拖拽排序**。
- 固定在前/后的列不参与“中间列”的拖拽排序（避免与固定列逻辑冲突）。

### 1.6 技术约定（组件与插槽）

- 统一使用 `CrmDataTable` 承载该能力。
- 启用列设置的必要条件：
  - `column-layout-key="xxx"`：列表唯一键（建议 `模块-页面`，如 `rfq-item-list-main`）
  - `:columns="columns"`：声明式列定义
  - `:show-column-settings="false"`：关闭表格顶部入口（本规范要求入口在底栏）
- 单元格自定义渲染使用具名插槽：`#col-<key>`
- 表头自定义渲染使用具名插槽：`#col-<key>-header`
- 从外部触发打开抽屉：`dataTableRef.openColumnSettings()`

> 参考实现：
>
> - 组件：`CRM.Web/src/components/CrmDataTable.vue`
> - 持久化：`CRM.Web/src/composables/usePersistedTableColumns.ts`

### 1.7 持久化范围

- 当前实现为 **localStorage 本机持久化**：
  - 键：`crm-table-columns:v1:<columnLayoutKey>`
  - 内容：`middleOrder` + `hiddenKeys`
- **换浏览器/换设备不跟随**；若需账号级同步，应另行对接后端存储（不在本规范范围内）。
