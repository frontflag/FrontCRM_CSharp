# 业务列表规范

本规范沉淀所有「业务列表页」的通用 UI 与交互约定，作为新页面开发与代码评审的统一基线。

## 1. 列表表头（列设置：显示/隐藏/顺序/持久化）

### 1.1 目标

- **用户可自定义列顺序**（拖拽）
- **用户可自定义列显示/隐藏**（勾选）
- **设置需持久化**，下次进入仍生效
- **可一键恢复默认**

### 1.2 入口位置（重要）

- **入口不占用列表上方额外一行**（避免浪费纵向空间）。
- 入口放在**列表底栏左侧**（与分页同一行），并与右侧分页控件**顶部对齐**。
- 入口按钮仅显示**图标**（齿轮），不显示“列设置”文字；“列设置”通过 **Tip/Tooltip** 展示。

> 参考实现：`CRM.Web/src/views/RFQ/RFQItemList.vue`（需求明细）

### 1.3 底栏布局规则（间隔区）

- “列设置”按钮与右侧**空白间隔区域**（Spacer）之间，**可插入「行高密度」切换**（见第 2 章）；插入时建议从左到右为：**列设置（齿轮）→ 行高密度 → Spacer → 其他按钮**。
- Spacer 右侧开始摆放其余底栏按钮（与分页同排对齐约定不变）。
- “复选篮子”入口与相关按钮按同一逻辑放置（从 Spacer 右侧开始）。

> 参考实现：`RFQItemList.vue` 的 `.table-footer-bar` / `.table-footer-left` / `.table-footer-spacer`；行高密度与列设置同排见 `CustomerList.vue` 底栏。

### 1.4 列设置交互（抽屉）

- 点击齿轮打开右侧抽屉「列显示与顺序」。
- 抽屉内：
  - 勾选控制显示/隐藏
  - 拖拽控制顺序（仅对“中间列”生效）
  - “恢复默认”一键重置为页面初始列状态
- 文案提示需说明：设置保存在本机（localStorage），下次打开仍有效。

### 1.5 固定列与不可配置列

- **勾选列**（`type=selection`）必须固定在最前（pinned start），且 **不可隐藏**。
- **操作列**（右侧固定列，通常 `fixed='right'` 且 `class-name='op-col'`）必须固定在最后（pinned end），且 **不可隐藏**、**不可拖拽排序**。
- 固定在前/后的列不参与“中间列”的拖拽排序（避免与固定列逻辑冲突）。

### 1.6 技术约定（组件与插槽）

- 统一使用 `CrmDataTable` 承载该能力。
- 启用列设置的必要条件：
  - `column-layout-key="xxx"`：列表唯一键（建议 `模块-页面`，如 `rfq-item-list-main`）
  - `:columns="columns"`：声明式列定义
  - `:show-column-settings="false"`：关闭表格顶部入口（本规范要求入口在底栏）
- 单元格自定义渲染使用具名插槽：`#col-<key>`
- 表头自定义渲染使用具名插槽：`#col-<key>-header`
- 从外部触发打开抽屉：`dataTableRef.openColumnSettings()`

> 参考实现：
>
> - 组件：`CRM.Web/src/components/CrmDataTable.vue`
> - 持久化：`CRM.Web/src/composables/usePersistedTableColumns.ts`

### 1.7 持久化范围

- 当前实现为 **localStorage 本机持久化**：
  - 键：`crm-table-columns:v1:<columnLayoutKey>`
  - 内容：`middleOrder` + `hiddenKeys`
- **换浏览器/换设备不跟随**；若需账号级同步，应另行对接后端存储（不在本规范范围内）。

## 2. 列表行高密度（紧密 / 适中）

用于在业务列表中切换**数据行**视觉松紧；实现以 `CrmDataTable` 为准。**表头高度不参与本功能**（表头仍走全站表格表头样式）。**未使用 `CrmDataTable` 的裸 `el-table` 不在本章约定范围内**。

### 2.1 术语与数值定义

- **行高（本规范含义）**：仅指数据格 **`td.el-table__cell` 的垂直内边距之和**，即 `padding-top` + `padding-bottom`。不改变表头 `th` 的 padding。
- **紧密（compact）**：上述垂直内边距之和为 **10px**（即上下各 **5px**，由全局变量换算）。
- **适中（medium）**：上述垂直内边距之和为 **22px**（即上下各 **11px**）。
- **水平内边距**（数据格左右 padding，与垂直独立可调）：紧密默认 **10px**（左右各 5px 量级由变量控制）、适中默认 **14px**（左右各 7px 量级由变量控制）。

全局变量与样式入口（便于统一改数）：

- `CRM.Web/src/assets/styles/variables.scss`：`$crm-list-row-density-compact-cell-padding-y-total`、`$crm-list-row-density-medium-cell-padding-y-total` 及对应 `*-padding-x`。
- `CRM.Web/src/assets/styles/crm-list-table-row-density.scss`：在 `CrmDataTable` 根节点修饰类下覆盖 **主表体与右侧固定列副本** 的 `td`，保证同列对齐。

### 2.2 默认与持久化

- **默认状态**：**紧密**（无本地记录或键无效时）。
- **持久化**：`localStorage`，键前缀 **`crm-table-row-density:v1:`**，完整键为 **`crm-table-row-density:v1:<storageKey>`**，取值 **`compact`** 或 **`medium`**。
- **storageKey 来源**：优先使用 **`column-layout-key`**（与列布局共用同一列表页唯一标识）；若无 `column-layout-key`，可显式传 **`row-density-storage-key`**。
- **按列表页独立记忆**：不同 `storageKey` 互不覆盖；用户切换后保持直至再次切换。

读写工具：`CRM.Web/src/utils/crmTableRowDensityStorage.ts`。

### 2.3 底栏入口与视觉

- **展示条件**：存在有效 `storageKey`、未将 **`show-row-density-toggle`** 设为 `false`，且页面向 **`density-toggle-anchor-el`** 传入**非空**锚点（见 2.4）时，显示两个切换入口。
- **位置**：与「列设置」**同一行、底栏左侧**；建议在列设置**右侧**紧邻排列（与 `CustomerList.vue` / `VendorList.vue` 一致）。通过 **`Teleport`** 挂到页脚占位节点（`ref` 绑定空 `span`）即可与分页、齿轮对齐。
- **样式**：与列设置齿轮一致——**`el-button` `link` `type="primary"`**，并加 **`list-settings-btn`** 图标按钮尺寸；两个按钮之间**无额外横向间距**（`margin`/`gap` 为 0）。
- **图标语义（当前约定）**：
  - **紧密**：**3 条横线**（表示更密）。
  - **适中**：**2 条横线**（表示相对疏）。
- **选中态**：当前模式为**主色不透明**；未选模式为**降低透明度**（muted），悬停可恢复可读。

Tooltip 文案：**「紧密」**、**「适中」**。

### 2.4 技术约定（`CrmDataTable`）

- 根节点自动增加 **`crm-items-table--density-compact`** 或 **`crm-items-table--density-medium`**，与 `crm-list-table-row-density.scss` 联动。
- 主要 props：
  - **`row-density-storage-key`**（可选）：无 `column-layout-key` 时用于持久化的独立键。
  - **`show-row-density-toggle`**：默认 `true`；设为 `false` 可隐藏底栏切换（仍按已存值渲染密度）。
  - **`density-toggle-anchor-el`**：页脚 **`HTMLElement | null`** 锚点（通常为空 `span` + `ref`）。**仅当传入非空锚点时**，底栏「紧密 / 适中」切换按钮才会通过 **Teleport** 渲染到该节点；不传或始终为 `null` 则不显示切换（行高仍按 `storageKey` 读取已保存密度应用）。
- 对外暴露（便于联调/测试）：**`rowDensity`**（ref）、**`setRowDensity('compact'|'medium')`**。

组件：`CRM.Web/src/components/CrmDataTable.vue`。

### 2.5 紧密模式下单元格内容单行化（必达）

**目标**：在 **紧密（compact）** 下，列表主表每一列的展示应尽量 **压缩为单行**——若该列在「适中」或默认结构下会把同一格内的信息拆成 **两行或多行**（例如主标题与副标题、主名称与编码上下堆叠），在紧密下 **应改为同一行内并排或连续展示**；过长内容用 **单行省略**（`text-overflow: ellipsis` + `overflow: hidden` + 父级 **`min-width: 0`**），必要时单元格或图标上提供 **`title` / Tooltip** 展示全文。

**与行高功能的关系**：`crm-list-table-row-density.scss` 只控制 **`td` 垂直/水平 padding**；**不自动**把多行 DOM 改成一行。是否单行化由 **各列表页** 在 **仅紧密态** 下通过样式或结构调整完成。

**实现约定**：

- 选择器必须挂在 **`crm-items-table--density-compact`**（由 `CrmDataTable` 根类提供）之下，使用 **`:deep(...)`** 穿透至表体单元格内层，**不得**在无前缀时全局改掉「适中」下的多行布局。
- 常用手段：容器 **`display: flex`**、`**flex-direction: row**`、`**flex-wrap: nowrap**`、子项 **`flex: 1 1 0; min-width: 0`**；副行/副文案取消 **`margin-top`** 并与主行 **`align-items: center`** 对齐；或 **`white-space: nowrap`** + **`text-overflow: ellipsis`**；多行块级文案可改为 **`display: -webkit-box; -webkit-line-clamp: 1`**（等价视觉单行）。

**例外（可不强制单行）**：

- 产品明确要求保留多行阅读的列（如备注全文、长描述预览），须在 PRD/评审中注明；若仍多行，应控制最大行数（如 `line-clamp`），避免在紧密 padding 下无限增高。
- 详情页内嵌表、弹窗/抽屉中的子表、调试页等非「标准业务列表主表」场景，按各页需求处理，**不强制**与本条完全一致。

**参考实现**（主名 + 副码/简称 两行 → 紧密下一行）：

- `CRM.Web/src/views/Customer/CustomerList.vue`：客户名称列「主名 + 状态图标」与「简称」横向并排。
- `CRM.Web/src/views/Vendor/VendorList.vue`：供应商名称列「主名 + 状态图标」与「编号」横向并排。

**新列表页自检**：接入行高密度后，在 **紧密** 下逐列检查是否存在「同一格内仍明显多行堆叠」；若有，按本条在 **`crm-items-table--density-compact`** 下补齐单行化样式或模板结构。
