## 列表操作列规范

### 目标
当列表行对应的“操作”按钮数量较多时，需要在不挤压业务数据列空间的前提下，提供：
1. “操作”列可收起/展开（列宽自适应）
2. 收起时行内按钮折叠为 `...` 下拉菜单
3. 展开时行内直接显示所有操作按钮（沿用现有按钮样式）

### 适用范围
适用于 Element Plus 的 `el-table`（通过项目封装的 `CrmDataTable` 亦可复用），列名为“操作”的末列（建议 `fixed="right"`）。

---

### 术语与状态
1. **展开态（Expanded）**：列宽为标准宽度；行内显示多个操作按钮
2. **收起态（Collapsed）**：列宽为窄列宽；行内仅显示一个 `...` 下拉菜单按钮
3. **列头切换**：列头按钮在展开态显示 `>`，在收起态显示 `<`
4. **默认状态**：列表进入页面时，“操作”列默认处于收起态（Collapsed）

---

### 交互行为（Column Interaction）
1. 在“操作”列的**列头**添加一个切换按钮（`type="button"`）
   - 展开态：按钮文案为 `>`
   - 收起态：按钮文案为 `<`
2. 点击列头切换按钮，切换展开/收起状态，并同步更新：
   - 列宽与 `min-width`
   - 行内展示方式（按钮列表 vs `...` 下拉菜单）

### 列宽要求（Width Rules）
1. 收起态列宽必须保证**能完整显示列头文字“操作”和 `<` 按钮**，避免列头被裁剪。
2. 推荐常量（参考当前项目的视觉尺寸，可按实际表格 padding 微调）：
   - 展开态宽度：`OP_COL_EXPANDED_WIDTH = 280`
   - 收起态最小宽度：`OP_COL_COLLAPSED_WIDTH >= 96`
3. 通过 `el-table-column` 动态绑定：
   - `:width="opColWidth"`
   - `:min-width="opColMinWidth"`

### 行内行为（Row Interaction）
1. **展开态**：在行内以原有方式纵向/横向展示所有可用操作按钮（例如：详情/编辑/申请采购/申请出库等）
2. **收起态**：
   - 行内仅显示一个 `...` 按钮
   - 点击 `...` 展开下拉菜单（`el-dropdown`）
   - 菜单项按“展开态的按钮”逐一对应为菜单项
3. 点击任意操作时需避免触发表格行级事件：
   - 对操作容器与按钮使用 `@click.stop`（必要时也加 `@dblclick.stop`）

---

### 样式规范（Style）
#### 1) 行内按钮容器
使用（或保持与全站现有样式一致）容器类名：
- `action-btns`：展开态按钮容器
- `op-col`：操作列的列级 class（用于覆盖 header/cell 的溢出、hover 行为等）

与项目一致的 hover 规则（推荐）：
- 收起/展开按钮在**默认状态下可保持隐藏/弱显示**，并在行 hover 时显示
- 示例规则（项目可复用现有实现）：
  - `action-btns { opacity: 0; transition: opacity 0.15s; }`
  - `tr:hover .action-btns { opacity: 1; }`

#### 2) 列头布局
列头建议使用 flex 布局：
- 左侧：列头文本（“操作”）
- 右侧：切换按钮（`<` / `>`）
并设置：
- `white-space: nowrap;`
- `overflow: hidden;`
以确保收起态下列头不被裁剪。

#### 3) `...` 按钮样式
类名建议：
- `op-more-trigger`

样式与可见性应遵循与 `action-btns` 相同的 hover 显隐逻辑（可复用 opacity 规则），确保收起态体验一致。

---

### 颜色与语义（Menu Item & Button Colors）
菜单项颜色与展开态按钮颜色保持一致（示例映射以项目现有语义色为准）：
1. `type="primary"` / 信息类：`$cyan-primary`（主青色）
2. `type="warning"` / 警告类：`$color-amber`（琥珀色）
3. `type="danger"` / 风险类：如项目已有 `$color-red-brown` 或对应 danger 配色（按现有规范/实现映射）

实现要点：
- 下拉菜单项使用与展开按钮同语义的 class 或颜色变量（例如：`op-more-item--primary` / `op-more-item--warning`）

---

### 推荐结构（Implementation Template）
1. `el-table-column` 设置：
   - `label="操作"`
   - `fixed="right"`
   - `align="center"`
   - `class-name="op-col"`
   - `label-class-name="op-col"`
   - 动态 `width/min-width`
2. 列头使用 `#header` slot：
   - `操作` + 切换按钮（`>` / `<`）
3. 行内容使用 `#default` slot：
   - 展开：`div.action-btns` + 多个 `el-button`
   - 收起：`el-dropdown` + `...` 触发器 + `el-dropdown-menu` + 对应操作项

---

### 可用性与约束（Usability & Constraints）
1. 切换按钮与 `...` 按钮均必须：
   - `type="button"`，避免在表单/容器内触发表单提交等副作用
2. 禁止点击操作区域触发行双击/行点击（需要 `stop`）
3. 收起态的列头与 `<` 按钮必须不被裁剪；若出现裁剪，优先调整：
   - `OP_COL_COLLAPSED_WIDTH` 的最小值
   - 列头 flex 布局（`space-between`、`nowrap`）

---

### 命名要求（Consistency）
为保证可维护性，建议使用以下命名（可按项目实际微调）：
- `opColExpanded`：展开/收起状态
- `toggleOpCol()`：列头切换方法
- `action-btns`：展开态按钮容器
- `op-col`：操作列 class
- `op-col-header` / `op-col-toggle-btn`：列头容器与切换按钮
- `op-more-trigger`：收起态 `...` 触发按钮
- `op-more-item--primary` / `op-more-item--warning`：菜单项颜色语义类
