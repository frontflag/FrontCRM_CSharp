## 列表操作列规范

### 强制项（必达）

- **「操作」列必须在列头提供「展开 / 收起」切换**：在列头区域提供可点击的切换控件（展开态显示 **`>`**、收起态显示 **`<`**，语义与 **§术语与状态** 一致），用于在 **收起态**（行内 `...` 菜单）与 **展开态**（行内平铺按钮）之间切换列宽与行内展示。
- **列头「操作」文案**：**不强制**在表头单元格内展示「操作」二字。为 **收窄操作列宽**，允许 **隐藏列标题文案**，列头 **仅保留 `<` / `>`**（居中或与设计稿一致）；列配置/列设置抽屉中仍可保留 `label="操作"` 等语义，便于用户识别该列。
- **豁免**：仅当 **PRD 或评审记录中显式写明豁免**（例如调试页、只读预览表、无行级操作的占位列）时，可不实现列头切换；豁免须在实现处注释中注明文档/评审指向。

### 目标

在符合 **强制项** 的前提下，在不挤压业务数据列空间的前提下，统一提供：

1. 「操作」列可收起/展开（列宽自适应）
2. 收起时行内按钮折叠为 `...` 下拉菜单
3. 展开时行内直接显示所有操作按钮（沿用现有按钮样式）

### 适用范围

适用于 Element Plus 的 `el-table`（通过项目封装的 `CrmDataTable` 亦可复用），语义为「操作」的末列（`label` 等仍可写「操作」；**表头是否显示该文案**见 **§强制项**）。建议 `fixed="right"`。**凡落入本范围的列表，须实现「强制项」**；未豁免的页面不得在「操作」列省略列头展开/收起切换。

---

### 术语与状态
1. **展开态（Expanded）**：列宽为标准宽度；行内显示多个操作按钮
2. **收起态（Collapsed）**：列宽为窄列宽；行内仅显示一个 `...` 下拉菜单按钮
3. **列头切换**：列头按钮在展开态显示 `>`，在收起态显示 `<`
4. **默认状态**：列表进入页面时，“操作”列默认处于收起态（Collapsed）

---

### 交互行为（Column Interaction）

1. **必须**在「操作」列的**列头**提供切换按钮（`type="button"`），即 **展开 / 收起** 控件：
   - 展开态：按钮文案为 `>`
   - 收起态：按钮文案为 `<`
2. 点击列头切换按钮，切换展开/收起状态，并同步更新：
   - 列宽与 `min-width`
   - 行内展示方式（按钮列表 vs `...` 下拉菜单）

### 列宽要求（Width Rules）
1. **列头含「操作」文案**（标准列表）：收起态列宽必须保证**能完整显示列头文字「操作」和 `<` 按钮**，避免列头被裁剪。
2. **列头仅图标**（隐藏列标题、高密度）：收起态只需保证 **`<` / `>`** 与行内 **`...`** 不被裁剪；列宽可显著小于标准列表。当前项目 **采购订单明细表** 参考值（可按业务按钮数量微调）：
   - 展开态：`width ≈ 173`，`min-width ≈ 160`
   - 收起态：`width` 与 `min-width` 可取 **约 43**（与仅图标列头、`...` 列对齐）
3. **标准列表**推荐常量（列头含「操作」文案时，可按实际表格 padding 微调）：
   - 展开态宽度：`OP_COL_EXPANDED_WIDTH = 280`
   - 收起态最小宽度：`OP_COL_COLLAPSED_WIDTH >= 96`
4. 通过 `el-table-column` 动态绑定：
   - `:width="opColWidth"`
   - `:min-width="opColMinWidth"`

### 行内行为（Row Interaction）
1. **展开态**：在行内以原有方式纵向/横向展示所有可用操作按钮（例如：详情/编辑/申请采购/申请出库等）
2. **收起态**：
   - 行内仅显示一个 `...` 按钮
   - 点击 `...` 展开下拉菜单（`el-dropdown`）
   - 菜单项按“展开态的按钮”逐一对应为菜单项
3. 点击任意操作时需避免触发表格行级事件：
   - 对操作容器与按钮使用 `@click.stop`（必要时也加 `@dblclick.stop`）

---

### 样式规范（Style）
#### 1) 行内按钮容器
使用（或保持与全站现有样式一致）容器类名：
- `action-btns`：展开态按钮容器
- `op-col`：操作列的列级 class（用于覆盖 header/cell 的溢出、hover 行为等）

与项目一致的 hover 规则（推荐）：
- 收起/展开按钮在**默认状态下可保持隐藏/弱显示**，并在行 hover 时显示
- 示例规则（项目可复用现有实现）：
  - `action-btns { opacity: 0; transition: opacity 0.15s; }`
  - `tr:hover .action-btns { opacity: 1; }`

#### 2) 列头布局

**默认（列头展示「操作」）**：列头**必须**使用 flex 布局（或与下述效果等价的布局），以保证「操作」文案与 **`<` / `>`** 切换按钮同排对齐：
- 左侧：列头文本（「操作」）
- 右侧：切换按钮（`<` / `>`）
并设置：
- `white-space: nowrap;`
- `overflow: hidden;`
以确保收起态下列头不被裁剪。

**高密度（隐藏列标题）**：列头可不展示「操作」文案，**仅展示居中（或设计约定对齐）的 `<` / `>`**；列宽按 **§列宽要求** 中「列头仅图标」一档收窄。

#### 3) `...` 按钮样式与热区反馈
类名建议（与全站 `crm-unified-list.scss` 一致）：
- `op-more-trigger`：`...` 本体
- `op-more-dropdown-trigger`：收起态整块可点热区（`el-dropdown` 触发包装，含 `padding` / `min-height` 等）

**可见性**：`...` 与展开态 `action-btns` 的 hover 规则保持一致（例如默认弱显、行或热区悬停时 `opacity: 1`），确保收起态体验一致。

**悬停背景（推荐/订单类高密度列表已实现）**：当鼠标移入 **操作列单元格**（如 `td.op-col:hover`）或移入 **`...` 热区**（`.op-more-dropdown-trigger:hover`）时，在 **`.op-more-dropdown-trigger`** 上增加 **浅色背景 + 圆角 + 阴影**，**背景外轮廓与热点一致**（即与全局定义的 `margin` / `padding` / `min-height` 所框定的可点区域一致，勿仅画在三个字符的像素框上）。暗色主题（`html[data-theme='dark']`）下宜略加深阴影、略调背景透明度，保证对比度可辨。

---

### 颜色与语义（Menu Item & Button Colors）
菜单项颜色与展开态按钮颜色保持一致（示例映射以项目现有语义色为准）：
1. `type="primary"` / 信息类：`$cyan-primary`（主青色）
2. `type="warning"` / 警告类：`$color-amber`（琥珀色）
3. `type="danger"` / 风险类：如项目已有 `$color-red-brown` 或对应 danger 配色（按现有规范/实现映射）

实现要点：
- 下拉菜单项使用与展开按钮同语义的 class 或颜色变量（例如：`op-more-item--primary` / `op-more-item--warning`）

---

### 推荐结构（Implementation Template）

实现须满足 **强制项**：列头 slot 内**不可省略** **`<` / `>`** 切换按钮；**可省略**列头内的「操作」二字（见 **§强制项** 与 **§列头布局 · 高密度**）。

1. `el-table-column` 设置：
   - `label="操作"`（供列设置等 UI 识别；表头展示可由 slot 覆盖为仅图标）
   - `fixed="right"`
   - `align="center"`
   - `class-name="op-col"`
   - `label-class-name="op-col"`
   - 动态 `width/min-width`
2. 列头使用 `#header` slot：
   - **默认**：`操作` + 切换按钮（`>` / `<`）
   - **高密度**：仅切换按钮（`>` / `<`），不渲染列头「操作」文案
3. 行内容使用 `#default` slot：
   - 展开：`div.action-btns` + 多个 `el-button`
   - 收起：`el-dropdown` + `...` 触发器 + `el-dropdown-menu` + 对应操作项

---

### 可用性与约束（Usability & Constraints）
1. 切换按钮与 `...` 按钮均必须：
   - `type="button"`，避免在表单/容器内触发表单提交等副作用
2. 禁止点击操作区域触发行双击/行点击（需要 `stop`）
3. 收起态下列头 **`<` / `>`** 与行内 **`...`** 必须不被裁剪。若列头仍展示「操作」文案，还须保证该文案不裁剪；若出现裁剪，优先调整：
   - `OP_COL_COLLAPSED_WIDTH` 的最小值
   - 列头 flex 布局（`space-between`、`nowrap`），或改为 **仅图标列头** 并收窄列宽

---

### 命名要求（Consistency）
为保证可维护性，建议使用以下命名（可按项目实际微调）：
- `opColExpanded`：展开/收起状态
- `toggleOpCol()`：列头切换方法
- `action-btns`：展开态按钮容器
- `op-col`：操作列 class
- `op-col-header` / `op-col-toggle-btn`：列头容器与切换按钮
- `op-more-trigger`：收起态 `...` 触发按钮
- `op-more-dropdown-trigger`：收起态下拉触发外层（热区与悬停背景载体）
- `op-more-item--primary` / `op-more-item--warning`：菜单项颜色语义类
