# 金额与币别代码显示范式（UI）

> 状态：**已落地参考实现** — 需求明细页底部「采购报价」单价阶梯列（`CRM.Web/src/views/RFQ/RFQItemList.vue`）。其他页面复用时请对齐本范式，必要时将样式类抽至全局（如 `assets` 公共样式）后再引用。

## 1. 目标

- 列表/明细中「带币别的金额」**不**使用前置货币符号（如 `¥`、`$`）。
- 统一为：**阿拉伯数字金额 + 半角空格 + 三位币别字母**，币别字母用颜色区分主要币种，其余归为扩展色。

## 2. 文本结构

| 组成 | 规则 |
|------|------|
| 金额 | 使用 `toLocaleString('zh-CN', { minimumFractionDigits: 2, maximumFractionDigits: 6 })`；无效或 `0` 时整格显示 **—**（不展示币别）。**例外**：详情页「基本信息」网格中带币别的额度/余额见 **§5.1**（可展示 `0.00` + 币别）。 |
| 间隔 | 金额与币别之间 **一个空格**（可用 `&nbsp;` 或独立 `span` 保证不换行断开）。 |
| 币别 | **三位大写英文字母**（`RMB`、`USD`、`EUR`、`HKD`、`JPY`、`GBP`），与后端/表单枚举数字一致。 |

**示例（语义）：**

- `3.45 RMB`（非 `¥3.45`）
- `4.50 USD`（非 `$4.50`）

## 3. 币别数字枚举 → 字母

与报价阶梯/明细一致（与当前 `dockTierCurrencyCode` 实现相同）：

| `currency`（数值） | 显示字母 |
|-------------------|----------|
| `1` 或未识别/缺省 | `RMB` |
| `2` | `USD` |
| `3` | `EUR` |
| `4` | `HKD` |
| `5` | `JPY` |
| `6` | `GBP` |

若后续增加新枚举，在本表与映射函数中同步增补；未列出的新码建议归入「其他 → 紫色」或单独定义颜色。

## 4. 币别字母颜色（当前定稿）

仅对 **三位字母** 上色；**金额数字** 的字色与字重见 **§5**（列表/阶梯）与 **§5.1**（详情基本信息网格），均 **不随币别变色**（币别字母单独上色）。

| 字母 | 语义 | 色值（HEX） | 参考类名（当前页内） |
|------|------|-------------|----------------------|
| `RMB` | 人民币 | `#ec4899`（粉） | `.dock-tier-ccy--rmb` |
| `USD` | 美元 | `#22c55e`（绿） | `.dock-tier-ccy--usd` |
| `EUR` | 欧元 | `#2563eb`（蓝） | `.dock-tier-ccy--eur` |
| `HKD` | 港币 | `#eab308`（黄） | `.dock-tier-ccy--hkd` |
| `JPY`、`GBP` 及未来其他 | 扩展 | `#a855f7`（紫） | `.dock-tier-ccy--purple` |

## 5. DOM 与样式约定（可复用骨架）

```html
<span class="金额类 tabular-nums">
  <span class="金额-整数">3</span><span class="金额-小数">.45</span>
</span>
<span class="间隔">&nbsp;</span>
<span class="币别基类 币别--rmb">RMB</span>
```

- **金额**：`font-variant-numeric: tabular-nums`，**数字整体 `font-weight: 700`（加粗）**；拆成 **整数段** 与 **小数点及小数段** 两个 `span` 时：两段字色与 **「报价数量」** 一致（当前全局样式为亮色 `#27292C`、暗色主题下为 `$text-primary`）。拆分须用 **`Intl.NumberFormat(...).formatToParts`**（与 `toLocaleString` 同 locale、同小数位规则），勿仅用 ASCII `.` 截取。
- **币别基类**：**常规字重**（`font-weight: 400`，不加粗）、`letter-spacing: 0.02em`，字号可与表格小字一致（当前实现 `12px`）；颜色仍按币种区分。

类名前缀为 `dock-tier-*`（全局样式见 `crm-quote-tier-dock.scss`）；若再抽象可改为例如 `.ccy-amt`、`.ccy-code`、`.ccy-code--rmb` 等，但 **颜色表与枚举映射须与本范式一致**。

### 5.1 详情页「基本信息」网格中的金额数字（定稿）

适用于 **业务详情页顶部信息卡片**内与「账期、地区」等标量同一视觉层级的字段（当前参考：**客户详情**「信用额度」「账户余额」，`CRM.Web/src/views/Customer/CustomerDetail.vue`）。

与 **§5 列表/采购报价阶梯** 区分：此处 **不** 使用 `Space Mono`、**不** 使用金额加粗 `font-weight: 700`、**不** 使用阶梯金额专用色 `#27292C` / `$text-primary` 强对比样式。

| 项目 | 约定 |
|------|------|
| 金额数字 | 与同一块内普通 `.info-item .info-value` **完全一致**：`font-family: 'Noto Sans SC', sans-serif`；`font-size: 13px`；`font-weight: 400`；`color: $text-secondary`（随主题变量）。 |
| 金额格式 | 与列表总额一致可用 `formatTotalAmountNumber`（**固定 2 位小数**）；与接口字段归一化（如 `creditLimit ?? creditLine`、`balance ?? creditLineRemain`）由页面自行约定。 |
| 币别字母 | 仍为 **三位字母 + `.dock-tier-ccy*` 色**，与 §4 一致；**不加粗**。 |
| DOM 骨架 | 外层 `span.info-value`（勿再叠 `info-value--amount` 以免改回主色/中粗体）；内层 `span.amount-with-code`：`flex` 横向、`align-items: baseline`；金额 plain `span` + `span.dock-tier-ccy`。 |
| 负值 | 整格可加 `amount-negative` 等页面级修饰，金额数字 **继承** 该字色；币别字母是否一并变色由产品页决定（当前客户详情为整格着色）。 |

**示意（语义）：** 数字 `0.00` 与同一卡片内「账期(天)」的数字 `60` 为 **同一字体样式**；右侧 `USD` 仍为绿色 `.dock-tier-ccy--usd`。

## 6. 与旧式展示的边界

- **禁止**：`¥12.00`、`$4.50`、`US$4.50` 等符号前置或混合写法。
- **允许**：同一单元格内多行阶梯（每行各自「金额 + 空格 + 字母」），行距由列表规范约束。
