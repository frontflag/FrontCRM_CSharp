# 金额与币别代码显示范式（UI）

> 状态：**已落地参考实现** — 需求明细页底部「采购报价」单价阶梯列（`CRM.Web/src/views/RFQ/RFQItemList.vue`）。其他页面复用时请对齐本范式，必要时将样式类抽至全局（如 `assets` 公共样式）后再引用。

## 1. 目标

- 列表/明细中「带币别的金额」**不**使用前置货币符号（如 `¥`、`$`）。
- 统一为：**阿拉伯数字金额 + 半角空格 + 三位币别字母**，币别字母用颜色区分主要币种，其余归为扩展色。

## 2. 文本结构

| 组成 | 规则 |
|------|------|
| 金额 | 使用 `toLocaleString('zh-CN', { minimumFractionDigits: 2, maximumFractionDigits: 6 })`；无效或 `0` 时整格显示 **—**（不展示币别）。 |
| 间隔 | 金额与币别之间 **一个空格**（可用 `&nbsp;` 或独立 `span` 保证不换行断开）。 |
| 币别 | **三位大写英文字母**（`RMB`、`USD`、`EUR`、`HKD`、`JPY`、`GBP`），与后端/表单枚举数字一致。 |

**示例（语义）：**

- `3.45 RMB`（非 `¥3.45`）
- `4.50 USD`（非 `$4.50`）

## 3. 币别数字枚举 → 字母

与报价阶梯/明细一致（与当前 `dockTierCurrencyCode` 实现相同）：

| `currency`（数值） | 显示字母 |
|-------------------|----------|
| `1` 或未识别/缺省 | `RMB` |
| `2` | `USD` |
| `3` | `EUR` |
| `4` | `HKD` |
| `5` | `JPY` |
| `6` | `GBP` |

若后续增加新枚举，在本表与映射函数中同步增补；未列出的新码建议归入「其他 → 紫色」或单独定义颜色。

## 4. 币别字母颜色（当前定稿）

仅对 **三位字母** 上色；**金额数字** 使用表格默认正文字色（不随币别变色）。

| 字母 | 语义 | 色值（HEX） | 参考类名（当前页内） |
|------|------|-------------|----------------------|
| `RMB` | 人民币 | `#ec4899`（粉） | `.dock-tier-ccy--rmb` |
| `USD` | 美元 | `#22c55e`（绿） | `.dock-tier-ccy--usd` |
| `EUR` | 欧元 | `#2563eb`（蓝） | `.dock-tier-ccy--eur` |
| `HKD` | 港币 | `#eab308`（黄） | `.dock-tier-ccy--hkd` |
| `JPY`、`GBP` 及未来其他 | 扩展 | `#a855f7`（紫） | `.dock-tier-ccy--purple` |

## 5. DOM 与样式约定（可复用骨架）

```html
<span class="金额类 tabular-nums">
  <span class="金额-整数">3</span><span class="金额-小数">.45</span>
</span>
<span class="间隔">&nbsp;</span>
<span class="币别基类 币别--rmb">RMB</span>
```

- **金额**：`font-variant-numeric: tabular-nums`，**数字整体 `font-weight: 700`（加粗）**；拆成 **整数段** 与 **小数点及小数段** 两个 `span` 时：两段字色与 **「报价数量」** 一致（当前全局样式为亮色 `#27292C`、暗色主题下为 `$text-primary`）。拆分须用 **`Intl.NumberFormat(...).formatToParts`**（与 `toLocaleString` 同 locale、同小数位规则），勿仅用 ASCII `.` 截取。
- **币别基类**：**常规字重**（`font-weight: 400`，不加粗）、`letter-spacing: 0.02em`，字号可与表格小字一致（当前实现 `12px`）；颜色仍按币种区分。

类名前缀为 `dock-tier-*`（全局样式见 `crm-quote-tier-dock.scss`）；若再抽象可改为例如 `.ccy-amt`、`.ccy-code`、`.ccy-code--rmb` 等，但 **颜色表与枚举映射须与本范式一致**。

## 6. 与旧式展示的边界

- **禁止**：`¥12.00`、`$4.50`、`US$4.50` 等符号前置或混合写法。
- **允许**：同一单元格内多行阶梯（每行各自「金额 + 空格 + 字母」），行距由列表规范约束。
