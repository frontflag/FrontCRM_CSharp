### 客户板块全部功能（后端 + 前端合并视角）

#### 一、客户主档管理

- **客户列表**
  - 条件搜索：客户编号/名称/联系人/电话（关键字）、客户类型、客户等级、行业、地区、启用状态、排序字段/方向。
  - 分页展示：页码、每页条数、总数。
  - 列出字段：客户编号、名称、类型、等级、主联系人、主联系人手机、地区、信用额度、账户余额、启用状态。
  - 行级操作：查看详情、编辑、删除（软删到回收站）、快速启用/停用。

- **客户创建 / 编辑**
  - 基本信息：编号（自动生成，编辑禁止修改）、名称、简称、类型、等级、行业、统一社会信用代码、所属业务员、地区（省市区级联）、备注。
  - 财务信息：信用额度、账期（天）、结算货币、税率、发票类型。
  - 联系人信息：在编辑表单中增删改联系人（姓名、性别、手机、邮箱、部门、职位、电话、是否默认）。
  - 保存草稿 / 保存并提交（前端区分状态，后端统一保存接口）。

- **客户详情**
  - 顶部信息卡：客户编号、名称、启用/停用状态、黑名单标记、客户等级。
  - 基本信息卡片：类型、统一社会信用代码、行业、地区、所属业务员、信用额度、账期、余额、创建时间、更新时间。
  - 顶部操作按钮：编辑客户、加入/解除黑名单、删除客户（进入回收站）、创建报价、创建订单。

#### 二、客户启用 / 停用 & 黑名单

- **启用 / 停用**
  - 后端接口：`POST /api/v1/customers/{id}/activate`、`POST /api/v1/customers/{id}/deactivate`。
  - 前端列表开关、详情顶部按钮联动，支持一键启用/停用客户。

- **客户黑名单**
  - 后端接口（通过前端 API 可见）：
    - 加入黑名单：`POST /api/v1/customers/{id}/blacklist`（带原因）。
    - 解除黑名单：`DELETE /api/v1/customers/{id}/blacklist`。
    - 黑名单列表：`GET /api/v1/customers/blacklist?PageIndex=&PageSize=&Keyword=...`。
  - 前端功能：
    - 客户详情页顶部“加入黑名单 / 解除黑名单”按钮，页面顶部显示黑名单标签（`blackList` 标志）。
    - 独立黑名单列表页（通过 `customerApi.getBlacklist`），支持分页和关键字查询。

#### 三、客户软删除与回收站

- **软删除 / 回收站**
  - 删除客户：`DELETE /api/v1/customers/{id}`，前端可传删除原因，后端执行软删除，将客户放入“回收站”。
  - 回收站列表：`GET /api/v1/customers/recycle-bin?PageIndex=&PageSize=&Keyword=`。
  - 恢复客户：`POST /api/v1/customers/{id}/restore`，从回收站恢复为正常客户。
  - 前端：
    - 列表页“删除客户”操作，调用 `customerApi.deleteCustomer(id, reason)`。
    - 回收站页面 `CustomerRecycleBin.vue`：展示已删除客户，支持搜索与恢复。

#### 四、客户联系人管理

- **后端接口**
  - `GET /api/v1/customers/{customerId}/contacts`：获取指定客户的联系人列表。
  - `POST /api/v1/customers/{customerId}/contacts`：创建联系人。
  - `PUT /api/v1/contacts/{contactId}`：更新联系人。
  - `DELETE /api/v1/contacts/{contactId}`：删除联系人。
  - `POST /api/v1/contacts/{contactId}/set-default`：设置默认联系人。

- **前端功能**
  - 客户编辑页内嵌联系人子表：支持增加/编辑/删除联系人，设为默认联系人。
  - 客户详情页“联系人”Tab：
    - 列表展示：姓名、性别、部门、职位、手机、邮箱、是否默认。
    - 操作：新增联系人（弹窗）、编辑、删除、设置默认。
  - 通用弹窗组件：`Customer/components/ContactDialog.vue` 用于创建/编辑联系人。

#### 五、客户地址管理

- **后端接口**
  - `GET /api/v1/customers/{customerId}/addresses`：获取地址列表。
  - `POST /api/v1/customers/{customerId}/addresses`：新增地址。
  - `PUT /api/v1/addresses/{addressId}`：更新地址。
  - `DELETE /api/v1/addresses/{addressId}`：删除地址。
  - `POST /api/v1/addresses/{addressId}/set-default`：设置默认地址。

- **前端功能**
  - 客户详情页“地址信息”Tab：
    - 显示地址类型（收货/账单）、联系人、电话、详细地址、是否默认。
    - 操作：新增地址、编辑地址、删除地址、设为默认地址。
  - 弹窗组件：`Customer/components/AddressDialog.vue` 负责地址表单增改。

#### 六、客户银行信息管理

- **后端接口**
  - `GET /api/v1/customers/{customerId}/banks`：获取银行账户列表。
  - `POST /api/v1/customers/{customerId}/banks`：新增银行账号。
  - `PUT /api/v1/banks/{bankId}`：更新银行账户。
  - `DELETE /api/v1/banks/{bankId}`：删除银行账户。
  - `POST /api/v1/banks/{bankId}/set-default`：设置默认银行账户。

- **前端功能**
  - 客户详情页“银行信息”Tab：
    - 显示账户名称、银行名称、支行、账号、币种、是否默认。
    - 操作：新增银行信息、编辑、删除、设为默认。
  - 弹窗组件：`Customer/components/BankDialog.vue` 处理银行信息表单。

#### 七、客户联系历史（跟进记录）

- **后端接口**
  - `GET /api/v1/customers/{customerId}/contact-history`：获取联系历史列表。
  - `POST /api/v1/customers/{customerId}/contact-history`：新增联系记录。
  - `PUT /api/v1/customers/{customerId}/contact-history/{historyId}`：更新联系记录。
  - `DELETE /api/v1/customers/{customerId}/contact-history/{historyId}`：删除联系记录。

- **前端功能**
  - 客户详情页中“联系历史”模块：
    - 展示联系类型、主题、内容、联系人、时间、下次跟进时间、结果等字段。
    - 支持新增、编辑、删除联系记录（通过 `customerApi.addContactHistory/updateContactHistory/deleteContactHistory`）。

#### 八、客户统计与仪表盘

- **后端**
  - `GET /api/v1/customers/statistics`：返回统计数据：
    - 客户总数、活跃客户数、本月新增客户数、总应收余额。
    - 按等级分布、按行业分布。

- **前端**
  - 客户列表顶部统计卡片：总客户数、活跃客户数、本月新增、应收余额。
  - Dashboard 中的客户相关统计展示（在仪表盘视图中使用同一接口）。

#### 九、审计与日志（操作日志 / 字段变更日志）

- **前端 API 封装**
  - `customerApi.getOperationLogs(customerId)`：获取客户操作日志列表。
  - `customerApi.getFieldChangeLogs(customerId)`：获取客户字段变更日志。

- **用途**
  - 在客户详情或独立页面展示谁在何时对哪个客户做了什么操作、改动了哪些字段（可进一步在 UI 上实现时间轴或表格视图）。

---

以上即**当前客户板块的完整功能清单**，包含：

- 客户主档 CRUD + 状态管理
- 黑名单管理
- 回收站（软删除/恢复）
- 联系人、地址、银行信息子模块
- 联系历史记录
- 统计报表
- 操作/字段变更日志

后续你若要在客户中集成“上传文档管理模块”，就可以基于这份功能图谱选择入口（例如：客户详情页新增“附件”Tab、客户联系人详情页挂载名片文档面板等）。
