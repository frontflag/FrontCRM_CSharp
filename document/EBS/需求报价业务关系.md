## 二、业务关系详解

### 1. 需求(RFQ)与需求明细(RFQ Item)

| 关系维度 | 说明 |
|---------|------|
| **关系类型** | 一对多 (1:N) |
| **业务含义** | 一个需求单可以包含多个物料明细 |
| **关联字段** | `rfqitem.RfqId` → `rfq.RfqId` |
| **级联操作** | 删除需求主表时，级联删除明细 |
| **业务场景** | 业务员创建需求时，一个需求单可包含多种物料 |

**示例**:
```
需求单 RFQ2024001
    ├── 明细1: 芯片 STM32F103, 数量1000, 目标价$2.5
    ├── 明细2: 电阻 10K, 数量5000, 目标价$0.01
    └── 明细3: 电容 100uF, 数量3000, 目标价$0.05
```

**数据表示例**:

**rfq 表**:
| RfqId | RfqCode | CustomerId | Status | TotalAmount |
|-------|---------|------------|--------|-------------|
| GUID1 | RFQ2024001 | CUST001 | 1 | 5000.00 |

**rfqitem 表**:
| ItemId | RfqId | MaterialId | Quantity | TargetPrice |
|--------|-------|------------|----------|-------------|
| ITEM01 | GUID1 | MAT001 | 1000 | 2.50 |
| ITEM02 | GUID1 | MAT002 | 5000 | 0.01 |
| ITEM03 | GUID1 | MAT003 | 3000 | 0.05 |

---

### 2. 需求明细(RFQ Item)与报价(Quote Item)

| 关系维度 | 说明 |
|---------|------|
| **关系类型** | 一对多 (1:N) |
| **业务含义** | 一个需求明细可以收到多个供应商的报价 |
| **关联字段** | `quoteitem.RfqItemId` → `rfqitem.ItemId` |
| **业务规则** | 多个供应商可以对同一物料进行报价 |
| **竞争机制** | 供应商之间进行价格竞争 |

**示例**:
```
需求明细: 芯片 STM32F103 (ITEM01)
    ├── 报价1: 供应商A, 价格$2.3, 交期7天
    ├── 报价2: 供应商B, 价格$2.4, 交期5天
    └── 报价3: 供应商C, 价格$2.2, 交期10天
```

**数据表示例**:

**quoteitem 表**:
| ItemId | QuoteId | RfqItemId | MaterialId | Price | Quantity | DeliveryDays |
|--------|---------|-----------|------------|-------|----------|--------------|
| QITEM1 | QUOTE01 | ITEM01 | MAT001 | 2.30 | 1000 | 7 |
| QITEM2 | QUOTE02 | ITEM01 | MAT001 | 2.40 | 1000 | 5 |
| QITEM3 | QUOTE03 | ITEM01 | MAT001 | 2.20 | 1000 | 10 |

---

### 3. 需求(RFQ)与报价主表(Quote)

| 关系维度 | 说明 |
|---------|------|
| **关系类型** | 一对多 (1:N) |
| **业务含义** | 一个需求可以收到多个供应商的完整报价单 |
| **关联字段** | `quote.RfqId` → `rfq.RfqId` |
| **业务规则** | 每个供应商针对需求生成一张报价单 |
| **完整性** | 报价单必须包含需求中所有物料的报价(可部分报价) |

**示例**:
```
需求 RFQ2024001
    ├── 报价单 QUOTE001 (供应商A - VENDOR001)
    │       ├── 芯片 STM32F103: $2.3
    │       ├── 电阻 10K: $0.009
    │       └── 电容 100uF: $0.048
    │
    └── 报价单 QUOTE002 (供应商B - VENDOR002)
            ├── 芯片 STM32F103: $2.4
            ├── 电阻 10K: $0.01
            └── 电容 100uF: $0.05
```

**数据表示例**:

**quote 表**:
| QuoteId | QuoteCode | RfqId | VendorId | Status | TotalAmount | ValidDate |
|---------|-----------|-------|----------|--------|-------------|-----------|
| QUOTE01 | Q2024001 | GUID1 | VENDOR001 | 1 | 4800.00 | 2024-12-31 |
| QUOTE02 | Q2024002 | GUID1 | VENDOR002 | 1 | 5100.00 | 2024-12-31 |

## 三、完整业务流程

### 1. 业务流程图

```
┌─────────────┐
│  创建需求    │
│  (业务员)    │
└──────┬──────┘
       │
       ▼
┌─────────────┐
│  需求主表    │◄─────────────────────────┐
│  (rfq)      │                          │
│  Status=0   │                          │
└──────┬──────┘                          │
       │                                  │
       ▼                                  │
┌─────────────┐     ┌─────────────┐      │
│  需求明细表  │     │  分配采购员  │      │
│ (rfqitem)   │────►│             │      │
│             │     │ 系统自动分配 │      │
└─────────────┘     └──────┬──────┘      │
                           │              │
                           ▼              │
                    ┌─────────────┐       │
                    │  采购员报价  │       │
                    │             │       │
                    └──────┬──────┘       │
                           │              │
                           ▼              │
                    ┌─────────────┐       │
                    │   报价主表   │       │
                    │   (quote)   │───────┘
                    │  Status=0   │ (报价后可再次修改需求)
                    └──────┬──────┘
                           │
                           ▼
                    ┌─────────────┐
                    │  报价明细表  │
                    │ (quoteitem) │
                    │             │
                    └─────────────┘
                           │
                           ▼
                    ┌─────────────┐
                    │  业务员选价  │
                    │             │
                    │ 选择最优报价 │
                    └──────┬──────┘
                           │
                           ▼
                    ┌─────────────┐
                    │  生成订单    │
                    │             │
                    │ 销售订单/采购 │
                    └─────────────┘
```

### 2. 业务流程步骤说明

| 步骤 | 操作人 | 操作内容 | 涉及表 | 状态变化 |
|-----|--------|---------|--------|---------|
| 1 | 业务员 | 创建需求主表和明细 | rfq, rfqitem | Status=0(待分配) |
| 2 | 系统 | 自动分配采购员 | rfqitempurchaseuser | Status=1(已分配) |
| 3 | 采购员 | 接收需求并查看 | rfqitem | 更新读取状态 |
| 4 | 采购员 | 向供应商询价 | - | 外部操作 |
| 5 | 采购员 | 录入供应商报价 | quote, quoteitem | Status=0(待确认) |
| 6 | 业务员 | 查看报价列表 | quote, quoteitem | - |
| 7 | 业务员 | 选择最优报价 | rfqitem | 标记最优报价 |
| 8 | 系统 | 生成销售/采购订单 | sellorder/purchaseorder | Status=5(已转订单) |

## 四、关键业务规则

### 1. 需求状态流转

| 状态值 | 状态名称 | 说明 | 可操作人 |
|-------|---------|------|---------|
| 0 | 待分配 | 需求刚创建，等待分配采购员 | 系统 |
| 1 | 已分配 | 已分配给采购员 | 采购员 |
| 2 | 报价中 | 采购员正在向供应商询价 | 采购员 |
| 3 | 已报价 | 已完成报价录入 | 业务员 |
| 4 | 已选价 | 业务员已选择报价 | 业务员 |
| 5 | 已转订单 | 已生成销售/采购订单 | 系统 |
| 6 | 已关闭 | 需求被关闭或取消 | 业务员/采购员 |

**状态流转图**:
```
待分配(0) → 已分配(1) → 报价中(2) → 已报价(3) → 已选价(4) → 已转订单(5)
    ↓           ↓            ↓            ↓            ↓
  已关闭(6)   已关闭(6)    已关闭(6)    已关闭(6)    已关闭(6)
```

### 2. 报价与需求的对应规则

| 规则 | 说明 | 业务场景 |
|-----|------|---------|
| **一对多** | 一个需求明细可对应多个报价(不同供应商) | 多家供应商竞争同一物料 |
| **部分报价** | 供应商可以只对需求中的部分物料报价 | 供应商只有部分物料库存 |
| **多次报价** | 同一供应商可以对同一需求多次报价 | 价格调整后重新报价 |
| **最优报价** | 系统可自动推荐价格最低的报价 | 帮助业务员快速决策 |
| **有效期** | 报价有有效期限 | 超过有效期需重新询价 |

### 3. 数据完整性约束

| 约束 | 说明 | 触发条件 |
|-----|------|---------|
| 需求删除级联 | 删除需求主表时，级联删除明细和关联的报价 | 删除rfq记录时 |
| 报价依赖 | 报价必须关联有效的需求明细 | 插入quoteitem时 |
| 状态控制 | 已转订单的需求不允许修改 | 更新rfq时检查Status |
| 唯一性约束 | 需求编码、报价编码必须唯一 | 插入/更新时 |
