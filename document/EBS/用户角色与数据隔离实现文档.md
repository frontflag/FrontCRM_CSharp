# 用户角色与数据隔离实现文档

## 一、概述

EBS系统采用基于**部门**和**角色**的权限模型，实现销售、采购等不同业务方向的数据隔离。

### 核心特点

| 特点 | 说明 |
|------|------|
| **部门为核心** | 用户归属部门，部门决定业务身份类型和数据权限范围 |
| **双向隔离** | 销售数据和采购数据完全隔离，互不可见 |
| **灵活配置** | 支持全部/自己/禁止三种数据权限范围 |
| **场景过滤** | 支持按业务场景进行细粒度数据过滤 |

### 2. 部门业务身份类型 (EBusinessIdentityType)

| 身份类型 | 值 | 说明 |
|---------|---|------|
| None | 0 | 无身份 |
| SalesMan | 1 | **业务员** - 销售方向 |
| Purchaser | 2 | **采购员** - 采购方向 |
| PurchaseAssistant | 3 | **采购助理** - 采购方向 |
| CustService | 4 | **客服** - 销售方向 |
| Finance | 5 | **财务** - 可查看两边数据 |
| Logistics | 6 | **物流** - 可查看两边数据 |

### 3. 数据权限范围 (EPermissionScope)

在部门表(`pst_department`)中定义了两个关键字段：

| 字段 | 说明 |
|------|------|
| **SaleDataScope** | 销售方向数据权限范围 |
| **PurchaseDataScope** | 采购方向数据权限范围 |

权限范围枚举值：

| 范围 | 值 | 说明 |
|------|---|------|
| **All** | 0 | **全部数据** - 可查看所有数据 |
| **Self** | 1 | **自己的数据** - 只能查看自己创建或归属的数据 |
| **Forbid** | 2 | **禁止访问** - 无法查看任何数据 |

## 三、数据隔离实现机制

### 1. 隔离原则

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                           数据隔离原则                                       │
└─────────────────────────────────────────────────────────────────────────────┘

    【销售方向】                              【采购方向】
    
    业务员/客服                               采购员/采购助理
         │                                        │
         │ 只能查看                               │ 只能查看
         ▼                                        ▼
    ┌─────────┐                              ┌─────────┐
    │ 销售订单 │                              │ 采购订单 │
    │ 销售明细 │                              │ 采购明细 │
    │ 客户资料 │                              │ 供应商资料│
    │ 销项发票 │                              │ 进项发票 │
    │ 出库数据 │                              │ 入库数据 │
    │ 收款记录 │                              │ 付款记录 │
    └─────────┘                              └─────────┘
         │                                        │
         │ 互相隔离                                │ 互相隔离
         └────────────────┬───────────────────────┘

                          ▼
                   ┌─────────────┐
                   │  财务/物流   │
                   │ 可查看两边  │
                   └─────────────┘
```

### 2. 数据隔离实现方式

#### 方式一：全局过滤器 (Global Filter)

```csharp
// TradeMainGlobalFilter.cs - 全局过滤器配置
public sealed class TradeMainGlobalFilter
{
    public static void Init()
    {
        // 报关明细权限全局过滤
        GlobalFilters.Register<TradeMainDbContext, DeclareCustomsItem>("GF-" + nameof(DeclareCustomsItem), (datas, usr, ctx) =>
        {
            // 1. 系统管理员查看全部
            if (usr.IsSysAdmin)
                return datas;
            
            // 2. 非员工用户禁止访问
            if (usr.UserType != EUserType.Employee)
                return datas.Where(t => false);
            
            // 3. 销售方向限制自己的数据范围
            if (usr.IsSaleFlag() && usr.BelongEmp.SaleDataScope != EPermissionScope.All)
            {
                var usrGrpIds = ctx.GroupUserTradeMain
                    .Where(t => usr.BelongEmp.ChildUserIds.Contains(t.UserId))
                    .Select(t => t.GroupId);
                return datas.Where(t => usrGrpIds.Any(p => p == t.SalesUserId));
            }
            
            return datas;
        });
    }
}
```

#### 方式二：服务层权限过滤方法

```csharp
// TradeMainBaseService.cs - 基础服务类提供权限过滤方法

/// <summary>
/// 权限过滤销售订单
/// </summary>
public IQueryable<SellOrder> FilterPermissionSellOrder(TradeMainDbContext dbContext, IQueryable<SellOrder> source)
{
    // 1. 客户用户只能查看自己的订单
    if (CurrentUser.UserType == EUserType.Customer)
    {
        return source.Where(t => t.CustomerId == CurrentUser.BelongClient.Id);
    }
    
    // 2. 供应商用户禁止查看销售订单
    if (CurrentUser.UserType == EUserType.Vendor)
    {
        return source.Where(s => false);
    }
    
    // 3. 无身份或禁止访问的返回空
    if (CurrentUser.BelongEmp.CurrentIdentityType == EBusinessIdentityType.None
        || CurrentUser.BelongEmp.SaleDataScope == EPermissionScope.Forbid)
    {
        return source.Where(s => false);
    }
    
    // 4. 系统管理员或全部权限的直接返回
    if (CurrentUser.IsSysAdmin 
        || CurrentUser.BelongEmp.SaleDataScope == EPermissionScope.All)
    {
        return source;
    }
    
    // 5. 业务员只能查看自己组的数据
    if (CurrentUser.IsSaleFlag())
    {
        var groups = dbContext.GroupUserTradeMain
            .Where(s => CurrentUser.BelongEmp.ChildUserIds.Contains(s.UserId));
        return source.Where(t => groups.Any(p => p.GroupId == t.SalesUserId));
    }
    
    return source.Where(t => false);
}

/// <summary>
/// 权限过滤采购订单
/// </summary>
public IQueryable<PurchaseOrder> FilterPermissionPurchaseOrder(TradeMainDbContext dbContext, IQueryable<PurchaseOrder> source)
{
    // 1. 供应商用户只能查看自己的订单
    if (CurrentUser.UserType == EUserType.Vendor)
    {
        return source.Where(t => t.VendorId == CurrentUser.BelongClient.Id);
    }
    
    // 2. 系统管理员或采购全部权限的直接返回
    if (CurrentUser.IsSysAdmin 
        || CurrentUser.BelongEmp.PurchaseDataScope == EPermissionScope.All)
    {
        return source;
    }
    
    // 3. 采购员只能查看自己的数据
    if (CurrentUser.IsPurchaseFlag())
    {
        return source.Where(t => t.PurchaseUserId == CurrentUser.Id);
    }
    
    return source.Where(t => false);
}
```

## 四、业务场景数据权限控制

### 1. 销售订单数据权限

```csharp
// 销售订单数据权限判断逻辑
public bool HasSellOrderDataPermission(SellOrder order)
{
    // 1. 系统管理员有全部权限
    if (CurrentUser.IsSysAdmin)
        return true;
    
    // 2. 销售方向全部权限
    if (CurrentUser.BelongEmp.SaleDataScope == EPermissionScope.All)
        return true;
    
    // 3. 业务员查看自己组的数据
    if (CurrentUser.IsSaleFlag())
    {
        var userGroupIds = GetUserGroupIds(CurrentUser.Id);
        return userGroupIds.Contains(order.SalesUserId);
    }
    
    // 4. 客服查看自己跟进的数据
    if (CurrentUser.IsCustServiceFlag())
    {
        return order.CustServiceUserId == CurrentUser.Id;
    }
    
    return false;
}
```

### 2. 采购订单数据权限

```csharp

public bool HasPurchaseOrderDataPermission(PurchaseOrder order)
{
    // 1. 系统管理员有全部权限
    if (CurrentUser.IsSysAdmin)
        return true;
    
    // 2. 采购方向全部权限

        return true;

    if (CurrentUser.IsPurchaseFlag())
    {

    }
    
    return false;
}
```

### 3. 库存数据权限

```csharp
// 库存数据权限判断逻辑
public bool HasStockDataPermission(StockItem stock)
{
    // 1. 系统管理员有全部权限
    if (CurrentUser.IsSysAdmin)
        return true;
    
    // 2. 销售或采购全部权限可查看
    if (CurrentUser.BelongEmp.SaleDataScope == EPermissionScope.All 
        || CurrentUser.BelongEmp.PurchaseDataScope == EPermissionScope.All)
        return true;
    
    // 3. 财务和物流可查看库存
    if (CurrentUser.IsFinanceFlag() || CurrentUser.IsLogisticsFlag())
        return true;
    
    // 4. 业务员只能查看自己销售订单关联的库存
    if (CurrentUser.IsSaleFlag())
    {
        return IsStockRelatedToSalesUser(stock.StockItemId, CurrentUser.Id);
    }
    
    // 5. 采购员只能查看自己采购订单关联的库存
    if (CurrentUser.IsPurchaseFlag())
    {
        return IsStockRelatedToPurchaseUser(stock.StockItemId, CurrentUser.Id);
    }
    
    return false;
}
```

## 七、关键代码实现

### 1. 判断用户身份类型

```csharp
public static class UserExtensions
{
    /// <summary>
    /// 是否是业务员
    /// </summary>
    public static bool IsSaleFlag(this SysUser user)
    {
        return user.BelongEmp?.CurrentIdentityType == EBusinessIdentityType.SalesMan
            || user.BelongEmp?.CurrentIdentityType == EBusinessIdentityType.CustService;
    }
    
    /// <summary>
    /// 是否是采购员
    /// </summary>
    public static bool IsPurchaseFlag(this SysUser user)
    {
        return user.BelongEmp?.CurrentIdentityType == EBusinessIdentityType.Purchaser
            || user.BelongEmp?.CurrentIdentityType == EBusinessIdentityType.PurchaseAssistant;
    }
    
    /// <summary>
    /// 是否是财务
    /// </summary>
    public static bool IsFinanceFlag(this SysUser user)
    {
        return user.BelongEmp?.CurrentIdentityType == EBusinessIdentityType.Finance;
    }
    
    /// <summary>
    /// 是否是物流
    /// </summary>
    public static bool IsLogisticsFlag(this SysUser user)
    {
        return user.BelongEmp?.CurrentIdentityType == EBusinessIdentityType.Logistics;
    }
}
```

### 2. 检查数据权限

```csharp
/// <summary>
/// 检查是否有销售订单数据权限
/// </summary>
public bool HasHaveSellOrderDataPermission(Guid sellOrderId)
{
    var sellOrder = GetSellOrder(sellOrderId);
    if (sellOrder == null)
        return false;
    
    // 系统管理员有全部权限
    if (CurrentUser.IsSysAdmin)
        return true;
    
    // 销售方向全部权限

    if (CurrentUser.IsSaleFlag())
    {
        var userGroups = GetUserGroups(CurrentUser.Id);
        return userGroups.Contains(sellOrder.SalesUserId);
    }
    
    return false;
}

    if (purchaseOrder == null)

    if (CurrentUser.IsSysAdmin)

    // 采购方向全部权限
    if (CurrentUser.BelongEmp.PurchaseDataScope == EPermissionScope.All)

    }
```

## 九、总结

EBS系统的用户角色和数据隔离实现具有以下特点：

1. **部门驱动**：数据权限主要由部门的 `SaleDataScope` 和 `PurchaseDataScope` 控制
2. **身份区分**：通过 `IdentityType` 明确区分业务员、采购员等不同业务身份
3. **双向隔离**：销售数据和采购数据完全隔离，通过权限范围控制访问
4. **多层过滤**：通过全局过滤器、服务层方法、数据过滤服务三层机制实现权限控制
5. **灵活配置**：支持全部/自己/禁止三种权限范围，可灵活配置
