# 入库、库存、出库业务与数据关系文档

## FrontCRM 与本文档的命名对照（避免歧义）

| 本文档（EBS / 概念） | FrontCRM 当前/目标 | 说明 |
|----------------------|-------------------|------|
| **StockItemV2**（入库行 1:1 的库存明细层） | 目标表 **`stockitem`** / 实体 **`StockItem`** | 数量与采销冗余的**事实层**；细则见 `document/System/库存设计与实现方案-StockItem版.md` |
| 文中「核心库存表」若指**按物料/仓等汇总的一行** | 表 **`stock`** / **`StockInfo`** | **分桶汇总**，由 `stockitem` 聚合刷新（目标） |
| **StockInV2** 等 | **`stockin`** / **`StockIn`** | 入库主表 |
| **StockOutItem.StockItemId** | 目标为指向 **`stockitem`**；当前代码多为 **`StockOutItem.StockId` → `StockInfo`** | 实现演进以 StockItem 版文档 P2–P3 为准 |

下文仍保留原 EBS 表名与示意图，便于对照历史需求；**以 System 目录下 StockItem 版方案为 FrontCRM 目标架构权威说明**。

## 二、核心库存字段说明

### StockItemV2 (库存表) 核心字段

| 字段名 | 数据类型 | 说明 | 计算公式 |
|--------|---------|------|---------|
| **Qty** | long | 入库数量 | 初始入库数量 |
| **QtyStockOut** | long | 已出库数量 | 累计出库数量 |
| **QtyOccupy** | long | 分拣占用量 | 正在分拣的数量 |
| **QtySales** | long | 业务占有量 | 已分配但未出库的数量 |
| **QtyRepertory** | long | 现有库存 | Qty - QtyStockOut |
| **QtyRepertoryAvailable** | long | 可用库存 | Qty - QtyStockOut - QtyOccupy - QtySales |

---

## 三、库存计算公式

### 1. 基础公式

```csharp
// 现有库存 = 入库数量 - 已出库数量
现有库存 = Qty - QtyStockOut

// 可用库存 = 入库数量 - 已出库数量 - 分拣占用 - 业务占有
可用库存 = Qty - QtyStockOut - QtyOccupy - QtySales
```

### 2. 库存校验公式

```csharp
// 出库前校验：可用库存是否足够
bool IsHasEnoughStockQty(long needQty) {
    return QtyRepertoryAvailable - QtySales - needQty >= 0;
}

// 实际库存校验
bool IsStockEnough(long needQty) {
    return QtyRepertory >= needQty;
}
```

## 六、核心代码方法

### 1. 入库增加库存

```csharp
/// <summary>
/// 添加入库并生成库存
/// </summary>
public void AddStockIn(AddStockInRequestDto requestDto)
{
    // 1. 创建入库主表 StockInV2
    var stockIn = new StockInV2 {
        Qty = requestDto.Qty,
        QtyStockOut = 0,
        // ... 其他字段
    };
    
    // 2. 创建入库明细 StockInItem
    var stockInItem = new StockInItem {
        StockInId = stockIn.StockInId,
        Qty = requestDto.Qty,
        // ... 其他字段
    };
    
    // 3. 生成库存 StockItemV2
    var stockItem = new StockItemV2 {
        StockInItemId = stockInItem.StockInItemId,
        Qty = requestDto.Qty,                    // 入库数量
        QtyStockOut = 0,                         // 已出库=0
        QtyOccupy = 0,                           // 分拣占用=0
        QtySales = 0,                            // 业务占有=0
        QtyRepertory = requestDto.Qty,           // 现有库存=入库数量
        QtyRepertoryAvailable = requestDto.Qty,  // 可用库存=入库数量
        // ... 其他字段
    };
}
```

### 2. 出库减少库存

```csharp
/// <summary>
/// 确认出库 - 扣减库存
/// </summary>
public void ConfirmStockOut(Guid stockOutId)
{
    var stockOutItems = GetStockOutItems(stockOutId);
    
    foreach (var item in stockOutItems)
    {
        var stockItem = GetStockItem(item.StockItemId);
        
        // 扣减库存
        stockItem.QtyStockOut += item.Qty;           // 增加已出库数量
        stockItem.QtyRepertory -= item.Qty;          // 减少现有库存
        stockItem.QtyRepertoryAvailable -= item.Qty; // 减少可用库存
        
        // 清除占用
        stockItem.QtyOccupy -= item.Qty;             // 清除分拣占用
        stockItem.QtySales -= item.Qty;              // 清除业务占有
        
        UpdateStockItem(stockItem);
    }
}
```

### 3. 库存预占 (分配库存给销售订单)

```csharp
/// <summary>
/// 保存库存与销售订单明细关系 (预占库存)
/// </summary>
public void AddStockAscription(string stockItemCode, long qty, Guid sellOrderItemId)
{
    var stockItem = GetStockItemByCode(stockItemCode);
    
    // 校验可用库存
    if (stockItem.QtyRepertoryAvailable - qty >= 0)
    {
        // 创建归属关系
        var relation = new StockAscriptionRelation {
            StockItemId = stockItem.Id,
            SellOrderItemId = sellOrderItemId,
            Qty = qty,
            Status = (short)StockAscriptionRelationStatus.Finish
        };
        
        // 扣减可用库存
        stockItem.QtySales += qty;                    // 增加业务占有
        stockItem.QtyRepertoryAvailable -= qty;       // 减少可用库存
        
        SaveChanges();
    }
}
```

## 七、库存状态流转

### 1. 库存数量变化流程

```
初始状态: Qty=1000, QtyStockOut=0, QtyRepertory=1000, QtyRepertoryAvailable=1000

步骤1: 入库完成
    Qty=1000, QtyStockOut=0, QtyRepertory=1000, QtyRepertoryAvailable=1000
    
步骤2: 销售订单预占300
    QtySales=300, QtyRepertoryAvailable=700
    
步骤3: 分拣占用200
    QtyOccupy=200, QtyRepertoryAvailable=500
    
步骤4: 出库确认200
    QtyStockOut=200, QtyRepertory=800
    QtySales=100, QtyOccupy=0, QtyRepertoryAvailable=700
    
最终状态:
    Qty=1000 (入库数不变)
    QtyStockOut=200 (已出库200)
    QtyRepertory=800 (现有库存800)
    QtyRepertoryAvailable=700 (可用700)
    QtySales=100 (待出库100)
    QtyOccupy=0 (无分拣占用)
```

## 八、关键业务规则

### 1. 入库规则

| 规则 | 说明 |
|------|------|
| 入库来源 | 采购入库、退货入库、调拨入库 |
| 质检要求 | 入库前必须经过质检 |
| 库存生成 | 质检通过后自动生成库存记录 |
| 成本记录 | 入库时记录采购成本 |

### 2. 出库规则

| 规则 | 说明 |
|------|------|
| 出库条件 | 可用库存必须 ≥ 出库数量 |
| 预占机制 | 销售订单确认后预占库存 |
| 分拣占用 | 分拣过程中占用库存 |
| FIFO原则 | 先进先出，优先使用早期库存 |

### 3. 库存校验规则

```csharp
// 1. 出库前校验
if (stockItem.QtyRepertoryAvailable < needQty) {
    throw new BusinessException("可用库存不足");
}

// 2. 预占校验
if (stockItem.QtyRepertoryAvailable - stockItem.QtySales < needQty) {
    throw new BusinessException("可用库存已被占用");
}

// 3. 实际库存校验
if (stockItem.QtyRepertory < needQty) {
    throw new BusinessException("实际库存不足");
}
```
