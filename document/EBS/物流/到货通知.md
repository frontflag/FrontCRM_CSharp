# 到货通知、质检、入库业务流程文档

## 一、概述

### 1.1 业务背景

到货通知、质检、入库是供应链管理中紧密关联的三个核心环节，构成了商品从供应商到仓库的完整流程。

### 1.2 核心概念

| 概念 | 说明 |
|------|------|
| **到货通知** | 通知仓库准备收货的单据 |
| **质检** | 对到货商品进行质量检验 |
| **入库** | 将质检合格的商品入库到仓库 |

### 1.3 整体业务流程图

```
采购订单明细 → 创建到货通知 → 确认到货 → 创建质检单 → 质检结果 → 生成入库
                                              ↓
                                    ┌─────────┼─────────┐
                                    ▼         ▼         ▼
                                全部通过   部分通过   全部拒收
                                    ↓         ↓         ↓
                                全部入库   部分入库   生成异常单
                                          部分拒收
```

## 二、状态枚举

### 2.1 到货通知状态 (EnumStockInNotifyStatus)

| 值 | 状态 | 说明 |
|----|------|------|
| -1 | **Fail** | 审核失败 |
| 1 | **NeedSubmit** | 待提交 |
| 2 | **NeedAudit** | 待审核 |
| 9 | **NotReceive** | 未到货 |
| 10 | **Receive** | 到货待检 |
| 20 | **HasQc** | 已质检 |
| 100 | **StockIn** | 已入库 |

### 2.2 质检状态 (EnumQCStatus)

| 值 | 状态 | 说明 |
|----|------|------|
| 1 | **QCReject** | 未通过 |
| 2 | **QCPartial** | 部分通过 |
| 100 | **QCPass** | 已通过 |

### 2.3 质检入库状态 (EnumQCStockInStatus)

| 值 | 状态 | 说明 |
|----|------|------|
| 1 | **NoStockIn** | 未入库 |
| 2 | **HaveStockIn** | 全部入库 |
| 3 | **PartStockIn** | 部分入库 |
| 4 | **Refuse** | 全部拒收 |

## 三、到货通知流程

### 3.1 创建到货通知

**触发条件**:
- 采购订单已审核
- 供应商已推送股份
- 管控料已上传EUS文件

**仓库分配逻辑**:
```csharp
var storageDef = currency == RMB ? "大陆仓" : "海外仓";
```

### 3.2 到货通知状态流转

```
NeedSubmit(1) → NeedAudit(2) → NotReceive(9) → Receive(10) → HasQc(20) → StockIn(100)
```

---

## 四、质检流程

### 4.1 创建质检单

**前置条件**:
1. 到货通知状态为"到货待检"
2. 供应商不在管控名单
3. 同一到货通知尚未完成质检

**状态判定**:
```csharp
if (hasQCPass && hasQCReject)
    Status = QCPartial;      // 部分通过
else if (hasQCPass)
    Status = QCPass;         // 全部通过
else if (hasQCReject)
    Status = QCReject;       // 全部拒收
```

### 4.2 质检结果处理

| 质检结果 | 入库状态 | 后续处理 |
|----------|----------|----------|
| 全部通过 | HaveStockIn | 全部入库 |
| 部分通过 | PartStockIn | 部分入库，部分拒收 |
| 全部拒收 | Refuse | 生成异常单 |

---

## 五、入库流程

### 5.1 质检生成入库

**触发条件**:
- 质检已完成
- 有通过的质检明细

**入库状态判定**:
```csharp
if (全部通过)      StockInStatus = HaveStockIn;
else if (部分通过) StockInStatus = PartStockIn;
else if (全部拒收) StockInStatus = Refuse;
```

### 5.2 入库后处理

1. 更新到货通知状态为"已入库"
2. 更新质检入库状态
3. 生成库存
4. 发送通知邮件

---

## 六、核心业务规则

### 6.1 数量校验

| 校验项 | 规则 |
|--------|------|
| 质检数量 | ≤ 到货通知预期数量 |
| 入库数量 | ≤ 质检通过数量 |
| 明细数量 | 不能为0 |

### 6.2 状态流转关系

| 单据 | 状态流转 |
|------|----------|
| 到货通知 | 到货待检(10) → 已质检(20) → 已入库(100) |
| 质检单 | 未入库(1) → 全部/部分入库(2/3) 或 拒收(4) |
