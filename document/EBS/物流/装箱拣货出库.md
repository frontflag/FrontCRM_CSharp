# 拣货、装箱、出库业务逻辑文档

## 一、概述

### 1.1 业务背景
拣货、装箱、出库是销售订单履约的核心环节，完成从库存分配、物料拣选、包装到最终出库发货的完整流程。

### 1.2 核心概念

| 概念 | 说明 |
|------|------|
| **出库通知 (StockOutNotify)** | 销售订单明细生成的出库指令，记录需出库的物料和数量 |
| **装箱单 (Packing)** | 拣货后生成的包装单据，包含装箱明细和客户信息 |
| **拣货** | 从库存中选择具体批次物料的过程 |
| **备货** | 完成拣货后，将物料准备就绪的状态 |
| **出库单 (StockOut)** | 最终的发货单据，记录实际出库信息 |

### 1.3 整体业务流程

```
销售订单确认 → 生成出库通知 → 创建装箱单 → 拣货 → 备货 → 出库 → 发货
     ↓               ↓              ↓          ↓       ↓       ↓
  订单审核      待装箱列表    选择库存批次   扫描拣货  确认备货  物流发货
```

## 二、状态枚举定义

### 2.1 装箱单状态 (EnumPackingStatus)

| 值 | 状态 | 说明 |
|----|------|------|
| **1** | UnAudited (新建) | 装箱单刚创建，未提交 |
| **2** | New (待审核) | 已提交待审核 |
| **9** | PassAudited (审核通过) | 审批通过 |
| **13** | SalesConfirm (业务确认) | 业务员确认 |
| **15** | CheckProduct (已拣货) | 完成拣货操作 |
| **16** | ProductReady (已备货) | 备货完成，等待出库 |
| **19** | ReadyStockOut (准备出库) | 准备出库状态 |
| **20** | Finish (已出库) | 出库完成 |
| **-1** | FailAudited (审核失败) | 审核未通过 |

### 2.2 订单出库状态 (EnumStockOutStatus)

| 值 | 状态 | 说明 |

| **2** | Partial (部分出库) | 部分数量已出库 |

### 2.3 出库通知类型 (EnumStockOutNotifyType)

| 值 | 类型 | 说明 |

| **10** | Normal (正常出库) | 正常销售出库 |

## 三、出库通知 (StockOutNotify)

### 3.1 生成时机
出库通知在以下场景自动生成：
- 销售订单审核通过后
- 销售订单供应商确认后
- 备货入库完成后（针对备货模式订单）

### 3.2 核心字段


## 五、拣货流程详解

### 5.1 拣货触发条件

拣货发生在以下场景：
1. **创建装箱单时** - 首次拣货选择库存
2. **编辑装箱单时** - 修改已拣货批次
3. **补拣货** - 库存不足时追加拣货

### 5.2 拣货数据源

**可用库存来源：**

```csharp
// 1. 库存归属关系表 - 匹配销售订单的库存
var listStockItemFirst = from sa in listStockAscriptionRelation
                         join s in listStockItemDB on sa.StockItemId equals s.Id
                         where sa.QtyRepertoryAvailable > 0
                         select new StockItem() { ... };

// 2. 直接库存 - 可用库存大于0的批次
var listStockItemThird = from sa in listStockAscriptionRelation
                         join s in listStockItemDB on sa.StockItemId equals s.Id
                         join pi in listPackItem on sa.Id equals pi.StockAscriptionId
                         where sa.QtyRepertoryAvailable == 0
                         select new StockItem() { ... };
```

### 5.3 拣货界面展示字段

| 字段 | 说明 | 示例 |
|------|------|------|
| **批次号** | 入库批次编码 | RK202403260001 |
| **仓库** | 所属仓库 | 深圳仓/香港仓 |
| **位置** | 库位信息 | A-01-02 |
| **原厂型号** | PN | STM32F103C8T6 |
| **品牌** | Brand | ST |
| **数量** | 可用库存 | 1000 |
| **单位** | 计量单位 | PCS |
| **生产日期** | DateCode | 2024-01 |
| **CO** | Country of Origin | CN |
| **质检状态** | QC状态 | 通过 |
| **入库时间** | StockInTime | 2024-03-20 |

### 5.4 拣货数量计算

```
拣货规则：
1. 单次拣货数量 ≤ 库存可用数量
2. 累计拣货数量 ≤ 出库通知需求数量
3. 支持多批次组合拣货

计算公式：
本次可拣货数量 = Min(库存可用数量, 需求数量 - 已拣货数量)
```

### 5.5 拣货操作方式

#### 方式一：界面勾选
1. 在库存列表中勾选需要拣货的批次
2. 输入本次拣货数量
3. 点击"添加"按钮加入拣货清单

#### 方式二：扫码拣货
1. 扫描物料标签条码
2. 系统自动匹配库存批次
3. 确认数量后添加

#### 方式三：自动分配（快速拣货）
1. 点击"自动分配"按钮
2. 系统按先进先出(FIFO)原则自动分配库存
3. 自动填充拣货数量和批次

## 六、备货流程详解

### 6.1 备货概念
备货是指完成拣货后，仓库人员将拣选出的物料进行实物准备、核对、包装，确认可以出库的状态。

### 6.2 备货操作

**入口**：装箱单详情 → 点击"备货"按钮

**操作步骤：**

```
步骤1: 确认拣货完成
   └─> 状态必须为"已拣货(15)"

步骤2: 实物核对
   └─> 核对物料型号、数量、批次
   └─> 检查物料外观质量

步骤3: 包装准备
   └─> 选择包装方式
   └─> 打印装箱标签

步骤4: 确认备货
   └─> 点击"确认备货"按钮
   └─> 状态变更为"已备货(16)"
```

### 6.3 备货校验规则

```csharp
// 只有已拣货状态才能备货
if (packing.Status != (short)EnumPackingStatus.CheckProduct)
    throw new BusinessException("只有已拣货的装箱单才能备货");

// 校验拣货数量与需求数量
if (totalPickQty < requiredQty)
    throw new BusinessException("拣货数量不足，无法备货");
```

## 七、出库流程详解

### 7.1 出库触发条件

出库发生在以下场景：
1. **备货完成** - 已备货状态的装箱单
2. **多个装箱单合并出库** - 同一客户、同一类型的装箱单可合并

### 7.2 出库操作步骤

**入口**：装箱单列表 → 选择已备货装箱单 → 点击"出库"

```
┌─────────────────────────────────────────────────────────────┐
│  步骤1: 选择装箱单                                            │
│  └─> 勾选状态为"已备货(16)"的装箱单                           │
│  └─> 支持多选合并出库（同一客户、同一类型）                    │
│                                                             │
│  步骤2: 填写出库信息                                          │
│  └─> 出库日期（默认当天）                                     │
│  └─> 快递单号（快递发货时）                                   │
│  └─> 签收单号（自提/送货时）                                  │
│                                                             │
│  步骤3: 校验检查                                              │
│  └─> 管控清单检查（客户是否在管控清单）                        │
│  └─> 付款方一致性检查                                        │
│  └─> 物料类型一致性检查                                      │
│                                                             │
│  步骤4: 确认出库                                              │
│  └─> 生成出库单                                              │
│  └─> 扣减库存                                                │
│  └─> 更新装箱单状态为"已出库(20)"                             │
│  └─> 更新销售订单出库状态                                     │
└─────────────────────────────────────────────────────────────┘
```

### 7.3 核心出库代码

```csharp
// StockOutService.Logistics.cs - MakeStockOutReady 方法
public Guid MakeStockOutReady(List<Guid> packingIds, DateTime stockOutDate, string expressNo = "", string signNo = "")
{
    // 1. 校验：只有已备货状态才能出库
    if (listPacking.Any(s => s.Status != (short)EnumPackingStatus.ProductReady))
        throw new BusinessException("只有已备货的装箱单才能出库");
    
    // 2. 校验：相同客户才能合并
    if (listPacking.GroupBy(s => s.CustomerId).Count() > 1)
        throw new BusinessException("相同客户的装箱单才能合并");
    
    // 3. 校验：相同类型才能合并
    if (listPacking.GroupBy(s => s.Type).Count() > 1)
        throw new BusinessException("相同类型的出库才能合并");
    
    // 4. 生成出库单
    var stockOut = new StockOut();
    stockOut.BillCode = GetSystemCode(typeof(StockOut).Name);
    stockOut.Status = (short)EnumStockOutStatusV2.New;
    stockOut.StockOutTime = stockOutDate;
    stockOut.Type = packingFirst.Type;
    
    // 5. 创建出库明细
    foreach (var packing in listPacking)
    {
        StockOutItem stockOutItem = new StockOutItem();
        stockOutItem.MainId = mianId;
        stockOutItem.Qty = packingItem.Qty;
        stockOutItem.StockItemId = packingItem.StockItemId;
        // ... 其他字段
    }
    
    // 6. 扣减库存
    DeductStockQty(listPackingItem);
    
    // 7. 更新装箱单状态
    packing.Status = (short)EnumPackingStatus.Finish;
    
    // 8. 发送出库消息
    SendStockOutMessage(stockOut);
}
```

### 7.4 出库后处理

出库完成后系统自动执行：
1. **扣减库存** - 减少对应批次的库存数量
2. **更新销售订单** - 更新订单出库状态（部分出库/出库完成）
3. **生成应收** - 触发应收账款生成
4. **发送通知** - 通知相关业务员和客户
5. **业务编号处理** - 生成业务编码关联

## 八、核心业务校验规则

### 8.1 数量校验

| 校验点 | 规则 |
|--------|------|
| 拣货数量 | 不能超过库存可用数量 |
| 装箱数量 | 不能超过出库通知需求数量 |
| 累计装箱 | 累计 ≤ 销售订单数量 |
| 出库数量 | 必须等于装箱数量 |

### 8.2 状态校验

| 操作 | 允许的状态 |
|------|------------|
| 编辑装箱单 | 新建、审核失败 |
| 提交审核 | 新建 |
| 拣货 | 审核通过、业务确认 |
| 备货 | 已拣货 |
| 出库 | 已备货 |

### 8.3 数据一致性校验

```csharp
// 1. 客户一致性
if (listPacking.GroupBy(s => s.CustomerId).Count() > 1)
    throw new BusinessException("相同客户的装箱单才能合并");

// 2. 类型一致性
if (listPacking.GroupBy(s => s.Type).Count() > 1)
    throw new BusinessException("相同类型的出库才能合并");

// 3. 付款方一致性
if (checkPayUnit > 1)
    throw new BusinessException("装箱单里面的出库通知的付款方只能允许一种！");

// 4. 物料类型一致性
if (checkMaterialType > 1)
    throw new BusinessException("装箱单里面的物料类型只能允许一种！");

// 5. 币别一致性（创建时）
if (listStockOutNotify.Select(s => s.Currency).Distinct().Count() > 1)
    throw new BusinessException("出库通知币别不一致！");
```

## 十二、业务特点总结

### 12.1 核心特点

1. **三单分离** - 出库通知、装箱单、出库单三层结构，支持灵活组合
2. **先进先出** - 库存拣货默认按FIFO原则
3. **多批次支持** - 同一物料支持多批次组合拣货
4. **合并出库** - 支持多个装箱单合并生成一张出库单
5. **状态流转** - 严格的状态控制，防止越权操作

### 12.2 注意事项

1. **库存冻结** - 拣货后库存会被冻结，需及时完成出库或取消
2. **数据一致性** - 客户、币别、物料类型必须保持一致
3. **时效性** - 拣货后需及时完成备货和出库
4. **权限控制** - 不同角色有不同的操作权限
