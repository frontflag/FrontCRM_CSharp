# ICVIP EBS系统报关业务逻辑和操作流程文档

## 1. 概述

报关模块是ICVIP EBS系统中处理跨境贸易报关业务的核心模块，主要负责：
- 报关单的创建、编辑、提交、审核
- 报关费用的自动计算（关税、增值税、代理费等）
- 报关状态的全流程跟踪
- 报关与销售订单、采购订单、库存等模块的关联管理
- 报关到货通知和报关入库的衔接

## 2. 业务流程

### 2.1 整体流程图

```
销售订单创建 → 采购订单创建 → 物料入库 → 出库通知单创建
                      ↓
                待报关清单筛选
                      ↓
                报关单创建（选择报关公司、填写报关信息）
                      ↓
                报关明细自动关联（销售、采购、库存）
                      ↓
                报关费用自动计算
                      ↓
                报关单保存/提交审核
                      ↓
                审核通过 → 报关中状态
                      ↓
                实际报关操作
                      ↓
                报关完成状态
                      ↓
                报关到货通知创建
                      ↓
                报关到货待检
                      ↓
                报关入库完成
```

### 2.2 详细操作步骤

#### 步骤1：待报关清单生成
- 从销售订单/出库通知单中筛选需要报关的物料
- 系统根据物料属性、产地等信息判断是否需要报关

#### 步骤2：报关单创建
- 选择报关公司（从系统维护的报关公司列表中选择）
- 填写报关单号、预计报关时间、报关合同号、入仓号等信息
- 设置汇率（手动填写，用于人民币计算）

#### 步骤3：报关明细生成
- 系统自动关联销售订单明细、采购订单明细和库存信息
- 每个报关明细项关联：`SellOrderItemId`、`PurchaseOrderItemId`、`StockItemId`、`StockInId`

#### 步骤4：报关费用计算
- 系统自动计算各项费用（详见第3章）
- 费用包括：报关公司货款、关税、增值税、代理费、杂费、商检费

#### 步骤5：报关单审核
- 提交给上级审核
- 审核通过后状态变为"报关中"
- 审核失败后状态变为"审核失败"

#### 步骤6：报关执行
- 报关公司进行实际报关操作
- 系统支持外部报关单同步功能

#### 步骤7：报关完成
- 报关完成后更新状态为"报关完成"
- 生成报关到货通知单

#### 步骤8：报关到货通知
- 创建报关到货通知单
- 状态变为"报关到货通知"

#### 步骤9：报关到货待检
- 报关物料到达仓库，等待质检
- 状态变为"报关到货待检"

#### 步骤10：报关入库
- 质检通过后完成报关入库
- 状态变为"报关入库"

## 3. 报关费用计算

### 3.1 费用构成

| 费用项 | 说明 | 计算公式 |
|--------|------|----------|
| **报关公司货款** | 采购成本折算为人民币 | `采购美金单价 × 汇率 × 数量` |
| **报关公司关税** | 关税费用 | `报关公司货款 × 关税税率` |
| **报关公司增值税** | 增值税费用 | `(报关公司货款 + 报关公司关税) × 增值税率(13%)` |
| **报关公司代理费** | 报关公司服务费 | `(报关公司货款 + 报关公司关税 + 报关公司增值税) × 报关公司费率` |
| **杂费** | 其他杂项费用 | 手动填写 |
| **商检费** | 商品检验费用 | 手动填写 |
| **价税总额** | 总费用合计 | `报关公司货款 + 报关公司增值税 + 报关公司代理费 + 杂费 + 报关公司关税` |
| **含税单价** | 单位成本 | `价税总额 ÷ 数量` |

### 3.2 详细计算公式

```csharp
// 采购美金价 = 原采购单价 × 系统汇率 × 采购系数
var usdPrice = GetDeclareCustomsPrice(cost, costCur, purchaseRatio);

// 报关公司货款 = 采购美金单价 × 汇率 × 数量
c.CustomsPaymentGoods = usdPrice * qty * exchangeRate;

// 报关公司关税 = 报关公司货款 × 关税税率
c.CustomsDuties = c.CustomsPaymentGoods * tariffRate;

// 报关公司增值税 = (报关公司货款 + 报关公司关税) × 增值税率
c.CustomsAddedvalueTax = (c.CustomsPaymentGoods + c.CustomsDuties) * vatRate;

// 报关公司代理费 = (报关公司货款 + 报关公司关税 + 报关公司增值税) × 报关公司费率
c.CustomsAgencyFee = (c.CustomsPaymentGoods + c.CustomsDuties + c.CustomsAddedvalueTax) * (customsRate - 1);

// 价税总额 = 报关公司货款 + 报关公司增值税 + 报关公司代理费 + 杂费 + 报关公司关税
c.TotalValueTax = c.CustomsPaymentGoods + c.CustomsAddedvalueTax + c.CustomsAgencyFee + c.OtherFee + c.CustomsDuties;

// 含税单价 = 价税总额 ÷ 数量
c.Price = c.TotalValueTax / qty;
```

### 3.3 费用场景枚举

系统定义了报关相关的费用场景：
- `EExpenseScene.DeclareTariff` - 报关关税
- `EExpenseScene.InspectionFee` - 商检费
- `EExpenseScene.Extra` - 杂费

## 4. 报关状态机

### 4.1 状态枚举定义

```csharp
public enum EnumDeclareCustomsStatus
{
    [Description("审核失败")] TobeArranged = -1,
    [Description("新建")] New = 1,
    [Description("待审核")] Adudi = 2,
    [Description("报关中")] DoingCustoms = 10,
    [Description("报关完成")] DoneCustoms = 11,
    [Description("报关到货通知")] NewStockInNotify = 12,
    [Description("报关到货待检")] CustomsReadyQC = 13,
    [Description("报关入库")] DoneStockIn = 14
}
```

### 4.2 状态流转规则

```
审核失败(TobeArranged) ←─┐
        ↑                 │
新建(New) → 待审核(Adudi) → 报关中(DoingCustoms) → 报关完成(DoneCustoms)
                                  ↓
                          报关到货通知(NewStockInNotify)
                                  ↓
                          报关到货待检(CustomsReadyQC)
                                  ↓
                          报关入库(DoneStockIn)
```

### 4.3 状态转换触发条件

| 状态转换 | 触发条件 |
|----------|----------|
| New → Adudi | 报关单提交审核 |
| Adudi → DoingCustoms | 审核通过 |
| Adudi → TobeArranged | 审核失败 |
| DoingCustoms → DoneCustoms | 报关完成 |
| DoneCustoms → NewStockInNotify | 创建报关到货通知 |
| NewStockInNotify → CustomsReadyQC | 报关到货待检 |
| CustomsReadyQC → DoneStockIn | 报关入库完成 |

## 8. 关联关系

### 8.1 与销售订单的关联

```
销售订单(SellOrder)
    ↓
销售订单明细(SellOrderItem) → 报关明细(DeclareCustomsItem)
```

- `DeclareCustomsItem.SellOrderItemId` 关联 `SellOrderItem.SellOrderItemId`
- 通过销售订单明细获取客户、业务员等信息

### 8.2 与采购订单的关联

```
采购订单明细(PurchaseOrderItem) ←───┘
    ↓
报关明细(DeclareCustomsItem)
```

- `DeclareCustomsItem.PurchaseOrderItemId` 关联 `PurchaseOrderItem.PurchaseOrderItemId`
- 通过采购订单明细获取采购成本、供应商等信息

### 8.3 与库存的关联

```
库存(StockItem) → 入库(StockIn)
    ↓
报关明细(DeclareCustomsItem)
```

- `DeclareCustomsItem.StockItemId` 关联 `StockItem.StockItemId`
- `DeclareCustomsItem.StockInId` 关联 `StockIn.StockInId`
- 通过库存获取物料实际存储信息

### 8.4 与报关公司的关联

```
报关公司(CustomsDef) ←───┘
    ↓
报关单(DeclareCustoms)
```

- `DeclareCustoms.CustomsDefId` 关联 `Customer.CustomerId`（报关公司作为特殊客户）
- 通过报关公司获取报关费率等信息

## 9. 业务规则

### 9.1 报关条件规则

1. **物料报关条件**：物料需要从境外进入境内时触发报关流程
2. **报关公司选择**：系统维护报关公司列表，选择时自动带入报关费率
3. **汇率管理**：支持手动填写汇率，用于人民币计算
4. **状态控制**：报关单状态严格按流程流转，不能跳过中间状态
5. **费用归属**：报关费用按物料明细归属到对应的销售订单和客户

### 9.2 权限控制规则

1. **业务员组**：`DeclareCustoms.SalesGroupId` 控制业务员访问权限
2. **采购员组**：`DeclareCustoms.PurchaseGroupId` 控制采购员访问权限
3. **多角色协作**：支持业务员、采购员、客服等多角色协同工作

### 9.3 数据一致性规则

1. **数量一致性**：报关数量不能超过库存可用数量
2. **费用一致性**：费用计算必须基于正确的汇率和税率
3. **状态一致性**：报关单状态与关联的装箱单状态保持一致

## 10. 外部报关单集成

### 10.1 集成功能

1. **外部报关单创建**：支持创建外部报关公司的报关单

## 12. 业务生命周期节点

### 12.1 报关业务节点服务

`CustomsClearanceBusinessNodeService`负责管理报关业务节点：

1. **节点生成**：根据报关单生成业务生命周期节点
2. **状态映射**：将报关状态映射为业务生命周期状态
3. **数量统计**：统计报关数量与销售数量的比例

### 12.2 状态映射关系

```csharp
protected EBusinessLifetimeNodeInstanceItemStatusStatus Map(EnumDeclareCustomsStatus status)
    => status switch
    {
        > EnumDeclareCustomsStatus.TobeArranged and < EnumDeclareCustomsStatus.DoingCustoms
```
