# 收款、付款、进项发票、销项发票业务逻辑文档

## 一、概述

### 1.1 业务背景
财务模块是EBS系统的核心模块之一，涵盖收款、付款、销项发票、进项发票四大核心业务，与销售、采购、仓储等模块紧密关联。

### 1.2 核心概念

| 概念 | 说明 |
|------|------|
| **收款** | 客户支付给公司的款项（应收账款回收） |
| **付款** | 公司支付给供应商的款项（应付账款支付） |
| **销项发票** | 公司开给客户的发票（收入确认） |
| **进项发票** | 供应商开给公司的发票（成本确认） |
| **应收** | 客户欠款（已发货未收款） |
| **应付** | 欠供应商款（已收货未付款） |
| **核销** | 收款与应收的匹配冲抵 |

### 1.3 整体业务流程

```
销售出库 → 生成应收 → 申请开票 → 开具销项发票 → 客户付款 → 收款核销
    ↓           ↓           ↓              ↓           ↓           ↓
  出库完成   应收账单    开票通知      电子发票     银行到账    应收清零

采购入库 → 生成应付 → 申请收票 → 收到进项发票 → 供应商付款 → 付款核销
    ↓           ↓           ↓              ↓           ↓           ↓
  入库完成   应付账单    收票通知      发票认证     审批付款    应付清零
```

|----|------|------|
| **1** | New (新建) | 付款单刚创建 |
| **2** | DAudit (待审核) | 已提交审批 |
| **10** | YPayment (应付款) | 审批通过待付款 |
| **16** | Paymenting (付款中) | 付款正在进行 |
| **20** | Finish (付款完成) | 付款已完成 |
| **-2** | Fail (审核失败) | 审批被拒绝 |

## 三、收款业务逻辑

### 3.1 业务流程

```
销售出库 → 生成应收(FinanceReceiptToBe) → 客户付款 → 创建收款单 → 核销应收 → 收款完成
```

### 3.2 应收生成

**生成时机**：出库完成后自动生成应收

**核心字段**：

| 字段 | 说明 |
|------|------|
| **BillCode** | 应收单号 |
| **CustomerId** | 客户ID |
| **SellOrderCode** | 销售订单号 |
| **PackingCode** | 装箱单号 |
| **PN** | 物料型号 |
| **Qty** | 数量 |
| **Price** | 单价 |
| **TotalAmount** | 总金额 |
| **VerificationDone** | 已核销金额 |
| **VerificationToBe** | 待核销金额 |
| **Currency** | 币别 |

### 3.3 收款流程

```csharp
// 1. 获取客户应收列表

        .Where(a => a.CustomerId == customerId &&

// 2. 创建收款单
public void CreateReceipt(AddReceiptModel model)
{
    // 创建收款主表
    var receipt = new FinanceReceipt();
    receipt.ReceiptCode = GetSystemCode("FinanceReceipt");
    receipt.CustomerId = model.CustomerId;
    receipt.ReceiptAmount = model.ReceiptAmount;
    receipt.Currency = model.Currency;
    
    // 创建收款明细
    foreach (var item in model.ReceiptItems)
    {
        var receiptItem = new FinanceReceiptItem();
        receiptItem.ReceiptToBeId = item.ReceiptToBeId;
        receiptItem.ReceiptAmount = item.Amount;
        // ...
    }
}

// 3. 核销应收
public void WriteOffReceipt(WriteOffModel model)
{
    // 更新应收核销状态
    receiptToBe.VerificationDone += writeOffAmount;
    receiptToBe.VerificationToBe -= writeOffAmount;
    
    // 更新收款核销状态
    receipt.WriteOffAmount += writeOffAmount;
    receipt.WaitWriteOffAmount -= writeOffAmount;
}
```

### 3.4 预收款

**业务场景**：客户提前支付货款，后续与出库应收进行核销

**流程**：
```
客户预付 → 创建预收款单(FinanCusAdvanceCollection) → 客户出库 → 核销预收 → 应收减少
```

```csharp
    SyncPaymentOrderInfoToHQIndustry(payment);

public void PaymentCompletedCallback(PaymentResult result)
{
    payment.Status = (short)FinancePaymentAudit.Finish;
    payment.PaymentDate = DateTime.Now;
    payment.PaymentAmount = payment.PaymentAmountToBe;
    
    // 核销应付
    WriteOffPaymentToBe(payment);
}
```

### 4.4 付款金额计算

```
付款总额 = 请款金额 - 抵扣金额 + 手续费 + 运费 + 杂费 - 尾差 - 其他抵扣
```

### 6.4 进项发票抵扣

```
可抵扣进项税额 = 发票金额 × 税率 / (1 + 税率)

例如：采购金额113元，税率13%
不含税金额 = 113 / 1.13 = 100元
进项税额 = 100 × 13% = 13元
```

```csharp
public void WriteOff(WriteOffModel model)
{

    var writeOff = new FinanceWriteOff();
```

## 十、核心业务规则

### 10.1 收款规则

1. **收款必须核销**：每笔收款必须与应收进行核销
2. **币别一致**：收款币别应与应收币别一致
3. **超额收款**：超额部分转为预收款
4. **部分收款**：支持多次收款核销同一笔应收

### 10.2 付款规则

1. **审批流程**：付款必须经过审批才能支付
2. **支付限额**：不同金额级别需要不同级别审批
3. **支付方式**：支持现金、银行转账、票据等多种方式
4. **费用分摊**：手续费、运费等费用可分摊到付款

### 10.3 发票规则

1. **发票与业务匹配**：发票必须与出库/入库明细匹配
2. **税率一致**：同一笔业务发票税率必须一致
3. **发票限额**：单张发票金额不能超过限额
4. **发票作废**：已开发票支持作废重开

### 10.4 核销规则

1. **先进先出**：按时间顺序优先核销早期应收/应付
2. **全额核销**：支持多笔收款核销一笔应收
3. **部分核销**：支持一笔收款核销多笔应收
4. **预收预付**：预收预付款可与应收应付进行核销
