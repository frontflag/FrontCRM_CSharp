# 报关与移库（方案 C）— 业务逻辑与实施方案

> 本文汇总报关模块与 **库存移库单（`StockTransfer` / `StockTransferItem`）** 的已定稿业务规则与实现要点；费用公式对齐《[EBS 报关业务逻辑](../EBS/物流/报关.md)》第 3 章。  
> **进口报关**一期落地；**出口**主表保留类型字段，业务与界面一期不处理。  
> **库存过账与虚拟出入库**：与 **手工移库** 共用 **《[移库共享内核_虚拟出入库实施方案](./移库共享内核_虚拟出入库实施方案.md)》** 中的 **`IInternalTransferPostingKernel`**；报关侧编排见 **本文 §13**；手工侧见《[手工移库功能实施方案](./手工移库功能实施方案.md)》。

---

## 1. 概述

### 1.1 业务目标

- 当 **出库通知** 表明货物流向为 **境内**，而实物在 **境外仓** 的在库商品需要进口报关时：经审批后生成 **待报关**，再形成 **报关单**；关务完成后通过 **移库单** 将数量从境外在库层扣减、在境内仓按现有分桶规则入账（**不**采用「整行修改 `StockItem.WarehouseId`」的迁仓方式）。
- **报关完成**与 **移库确认**采用 **方案 A：一步完成**——用户一次操作内完成报关状态落库与移库过账；失败则 **整单事务回滚** 并提示用户。
- 全链路满足 **1:1:1**：**一张出库通知 : 一张报关单 : 一张移库单**；**不支持分批移库**（一张报关单不拆多张移库单）。

### 1.2 方案 C 相对其他方式的取舍

| 方式 | 结论 |
|------|------|
| **直接改仓（改 `WarehouseId` + `StockAggregateId`）** | 部分数量时与 `StockInItemId` 唯一、汇总桶一致性冲突大，**不采用**。 |
| **标记为普通销售出库 + 采购入库** | 易污染对客/毛利统计，过滤面散，**不采用**。 |
| **方案 C：独立移库单 + 虚拟出/入库（调拨类型）** | 数量在**行级**表达；语义独立；**对称**落 **`stock_out`（`StockOutType=3` 调拨出库）+ `stock_in`（`StockInType=3` 调拨入库）** 作凭证，**库存数量仅在内核一次过账**；购销统计通过 **类型白名单（仅 `1`=销售/采购）** 与列表过滤收口；**采用**（与《移库共享内核_虚拟出入库实施方案》一致）。 |

### 1.3 核心单据与表（命名已定）

| 逻辑名 | 数据库表名（蛇形） | 说明 |
|--------|-------------------|------|
| 报关公司 | `customs_broker` | 独立主数据，**不**复用客户表。 |
| 报关主单 | `customsdeclarations`（实体名 `CustomsDeclarations` 等，以代码为准） | 报关单头。 |
| 报关明细 | `customsdeclarationitems` | 报关行。 |
| 移库主单（报关/客户侧） | **`stocktransfer_customers`**（首版迁移曾用名 `stocktransfer`） | 内部移库；一期仅 **报关进口迁库** 场景。 |
| 移库明细（报关/客户侧） | **`stocktransfer_item_customers`**（曾用名 `stocktransferitem`） | 与报关明细、源/目标在库层对应。 |

**业务流水号**：移库单复用 **`ModuleCodes` 新码段 `STF`**（与《[业务流水号编码规范](../PRD/规范/业务规范/业务流水号编码规范.md)》一致）。

## 2. 出库通知与待报关（前置链路）

### 2.1 触发条件

同时满足：

1. **`StockOutRequest.RegionType` = 境内**（表示货物流向/申报侧为境内；与 `WarehouseInfo` 地域枚举 **一致**）。
2. 该销售明细对应的 **在库商品位于境外仓**：判定与 **`WarehouseInfo`** 上地域为 **境外** 的定义一致。

### 2.2 审批与待报关生成

- **出库通知创建时**：若系统判断 **出库地域类型** 与 **仓库地域** **不吻合**（即触发报关场景），则生成 **待审批** 记录，审批类型为 **「出库通知报关」**。
- **审批通过后**：**自动生成「待报关」记录**（幂等：避免重复审批重复生成）。
- **快照**：在方案讨论中明确 **出库面向最终客户**、系统不单独处理「境外仓出库给报关公司」；待报关/报关明细与在库行的绑定以 **实现阶段** 与出库执行时序对齐的 **PRD 细则** 为准（创建时锁行或执行前锁行需在开发任务中单开子项）。

### 2.3 关系约束

- **出库通知 : 报关单 : 移库单 = 1 : 1 : 1**。
- **一张报关单** 可含 **多行** `CustomsDeclarationItems`；对应 **一张** `StockTransfer`、**多行** `StockTransferItem`（不违背「一张移库单」）。

## 3. 报关单业务规则

### 3.1 主表字段（至少）

- 报关单号、**类型（进口/出口）**（一期仅处理 **进口**）、申报日期、**海关状态**（枚举，**人工切换**）、**税费总额**。
- 与 **报关公司**、费用试算相关字段（汇率、税率等，与 EBS 第 3 章一致部分）。
- **内部状态机**（与海关状态分离）：**待报关 → 报关中 → 报关完成**，另加 **作废**；**不设「草稿」**。

### 3.2 海关状态与「报关完成」

- **海关状态**为关务事实侧进展（如：未申报、已放行、**已结关**等），**人工维护**。
- **仅当 海关状态 = 已结关** 时，才允许用户操作 **「报关完成」**（与移库一步触发的前置校验）。

### 3.3 作废规则

- **移库完成后**：报关单 **不可作废**（即用户选择「报关完成」且事务成功后，报关单进入不可作废状态；具体以状态 + 是否存在已确认移库单联合约束）。
- **草稿报关**：**不允许**生成移库单。
- **报关完成** 与 **移库** 为 **方案 A：一步完成**——**不**再单独保留「移库单草稿」给用户二次点确认；界面可用 **同一弹窗内两段勾选** 满足「报关完成」「移库确认」的 **二次确认** 与 **两条审计日志**（事件类型区分，如 `CustomsDeclarationCompleted`、`StockTransferConfirmed`）。

### 3.4 报关明细（`CustomsDeclarationItems`）

至少包括：商品明细、**HS 编码**、申报数量、申报单价、关税、增值税；以及费用侧与 EBS 第 3 章对齐的字段（货款、关税、增值税、代理费、杂费、商检费、价税总额、含税单价等，以定稿字段表为准）。

### 3.5 附件

报关单界面支持 **多个文件上传**（存储策略、大小限制、权限在实现任务中单列）。

### 3.6 费用公式（与 EBS 第 3 章一致）

| 费用项 | 计算公式 |
|--------|----------|
| 报关公司货款 | 采购美金单价 × 报关用汇率 × 数量 |
| 报关公司关税 | 报关公司货款 × 关税税率 |
| 报关公司增值税 | (报关公司货款 + 报关公司关税) × 增值税率（如 13%） |
| 报关公司代理费 | (货款 + 关税 + 增值税) × 费率；若系统存「1+费率」则用 ×(rate−1)，**二选一口径写死** |
| 杂费 / 商检费 | 手工录入 |
| 价税总额 | 货款 + 关税 + 增值税 + 代理费 + 杂费 + 商检费 |
| 含税单价 | 价税总额 ÷ 数量 |

## 4. 移库单（`StockTransfer` / `StockTransferItem`）

### 4.1 头表 `StockTransfer`

- **`BizScene`**：**一期仅 `CustomsImport`**；预留 **`Internal`（普通调拨）** 表结构，一期不开放业务。
- **`FromWarehouseId`、`ToWarehouseId`**：**均必填**。
- **`CustomsDeclarationId`**：**1:1** 绑定报关主单。
- **状态**：与方案 A 一致，移库在报关完成同一事务内 **直接确认（已过账）**；**移库完成后移库单不可作废**（回滚/冲销 **二期**）。

### 4.2 行表 `StockTransferItem`

| 字段 | 规则 |
|------|------|
| **`SourceStockItemId`** | **必填**，境外被扣减的在库层。 |
| **`CustomsDeclarationItemId`** | **必填**（与报关行 1:1 对应，冗余溯源）。 |
| **`StockOutRequestId`** | **必填**（冗余溯源）。 |
| **`TargetStockItemId`** | **过账成功后写入** 境内新生成的 `StockItem` 主键；过账前 **不允许** 改行、删行。 |

### 4.3 在库与分桶（境内入账）

- **境内目标 `StockInfo` 桶**：**严格复用**现有 `InventoryCenterService` 中与入库一致的分桶逻辑 **`StockMatchesInboundBucket`**，且 **`RegionType` = 境内**。
- **分桶键** **不包含** `MaterialId`（与现网一致；极端跨物料同键风险由业务约束承担，或在运营规范中规避）。
- **源 `StockItem` 部分数量**：**不拆行**；在源层上 **直接扣减数量**（与「一行一层部分移出」的实现约定一致）。

### 4.4 虚拟 `StockOut` / `StockIn`（调拨类型 3）

> **与旧版表述差异**：除原有 **虚拟入库** 外，**须增加对称的虚拟调拨出库单**（`StockOutType=3`），由 **共享内核** 在同一事务内写入；**不得**再调用 `StockOutService` / `StockInService` 的正式过账入口，**严禁双扣**。详见《移库共享内核_虚拟出入库实施方案》§一～§四。

- **虚拟调拨出库**：`stock_out` + `stock_out_item`，`StockOutType = **3**`（实体注释：调拨出库），`WarehouseId` = 源仓，`SourceId` = **`stocktransfer_customers` 主单主键**，`SourceCode` = 移库业务单号；`Remark` 建议固定前缀 **`【移库虚拟出库】`** + 单号。状态建议 **已生效**（如 `Status=2` 已出库语义），**不参与**拣货/执行出库流程。  
- **虚拟调拨入库**：`stock_in` + `stock_in_item`，`StockInType = **3**`（调拨入库），`WarehouseId` = 目标仓，`SourceId` / `SourceCode` 同上；`Remark` 建议 **`【移库虚拟入库】`**。  
- **境内新 `StockItem`**：因 **`stockitem.StockInItemId` 唯一**，新层 **必须** 绑定 **虚拟入库明细**；**`PurchaseOrderItem*` / `SellOrderItem*`** 等从 **源 `StockItem` 快照复制**；**`StockAggregateId`** 指向境内新匹配或新建的 **`stock`（`StockInfo`）** 汇总行。  
- **购销统计隔离**：凡「销售已出库」「采购已入库」累计须 **显式排除** `StockOutType=3` / `StockInType=3`（见共享内核 §五 checklist）。

## 5. 事务、失败回滚与用户提示

### 5.1 方案 A（一步完成）

- **「报关完成」** 提交：校验（含 **海关状态 = 已结关**、数量、桶、源层可扣等）→ 更新报关状态 → 创建并 **立即确认** `stocktransfer_customers` 及 **`stocktransfer_item_customers`** 行（或先写头再调内核，**`TransferHeaderId` 与 `SourceId` 对齐**）→ 调用 **`IInternalTransferPostingKernel.ExecuteAsync`**（`Kind=Customs`，多行则 `LineSnapshots` 一批）完成：**物理扣源 + 物理加目标 + 虚拟出 + 虚拟入 + `stockledger`** → 回写各 **`TargetStockItemId`** 等，**同一数据库事务**。  
- **禁止**：在 `CustomsDeclarationService` / Controller 内手写 SQL 改 `stock`/`stock_item` 数量；**唯一写口**为内核。

### 5.2 失败处理

- **任一步失败**（含 **内核过账 / 虚拟凭证写入失败**）：**整单回滚**；报关单内部状态回到 **「报关中」**。
- **对用户**：**合并为一条错误提示**（内部日志可区分失败点便于排查）。

### 5.3 幂等与重试

- 回滚后用户可再次点击「报关完成」；实现需 **幂等**（例如依赖「无已成功移库单」+ 状态机，避免重复流水/双单）。

## 6. `stockledger`（库存流水）

### 6.1 合并为一条 / 单一 `BizType`

- **每条 `stocktransfer_item_customers` 行** 对应 **一条** `stockledger` 记录（**不再**拆成出、入两条 ledger；**出/入凭证**由虚拟 `stock_out` / `stock_in` 承担）。
- **`BizType`**：统一为 **`STOCK_TRANSFER`**（或项目统一常量命名，长度受 `varchar(20)` 限制）；与手工移库的 **`MANUAL_TRANS`（或等价命名）** 区分，二者均由内核按 `Request.LedgerBizType` 写入。
- **`BizId`**：`StockTransferId`（即 `stocktransfer_customers.Id`）；**`BizLineId`**：`StockTransferItemId`，满足唯一索引 **`(BizType, BizId, BizLineId)`**。

### 6.2 扩表建议（在现有表上增量）

在保留原有列前提下，至少增加：

- **`from_warehouse_id` / `to_warehouse_id`**（或 `source_warehouse_id` / `target_warehouse_id`）：表达迁库方向；单列 `WarehouseId` 可与 **出方仓** 对齐以便旧逻辑按仓过滤，**以开发映射说明为准**。
- **`QtyIn` / `QtyOut`**：建议 **同值** = 移库数量（语义：从源仓出、入目标仓各该数量）；按仓汇总报表需用 **from/to** 两列拆分。
- **`create_by_user_id` varchar(36)**：与 `sellorder.create_by_user_id` 等 **新口径一致**；**`CreateUserId`（long）** 若无下游依赖可逐步不再写入。
- 可选：`customs_declaration_id`、`source_stock_item_id`、`target_stock_item_id` 等便于对账与排查。

### 6.3 实体与 EF

- 现有实体 **`InventoryLedger`** 映射表 **`stockledger`**；扩列后需更新 **`ApplicationDbContext`** 与 **`InventoryLedger`** 属性。

## 7. 统计、列表与独立入口

- **当前阶段**：**所有** 对外库存 **列表/汇总** 默认 **排除** 与 **`StockTransfer`（报关迁库）** 相关的数量变化（过滤条件与 **`BizType`** / 单据来源字段一致，避免遗漏）。
- **单独入口**：提供 **专门页面**（或菜单）查看 **移库单 / 报关迁库** 及关联流水，满足物流与审计对账。

## 8. 权限与审计

- **物流部门**：可操作 **报关单**、**移库单**（及与方案相关的前置审批通过后的待报关维护，以 RBAC 菜单为准）。
- **二次确认 + 操作日志**：在 **「报关完成」** 与 **「移库确认」** 语义上 **两处都要留痕**；**方案 A** 下推荐 **同一弹窗双勾选 + 两条审计事件**（或等价结构），满足审计要求。

## 11. 与共享内核对齐（报关移库实施要点）

| 项 | 说明 |
|----|------|
| **入口** | `CustomsDeclarationService.CompleteDeclarationAndTransferAsync`（当前仍可能 **占位抛错**；实装时替换为编排 + 内核）。 |
| **编排职责** | 校验报关内部状态、海关已结关、**1:1:1**、无重复移库单；写/更新 **`stocktransfer_customers`** 与 **`stocktransfer_item_customers`**（源 `SourceStockItemId`、报关行 ID、出库通知 ID 等按 §4.2）；组装 **`InternalTransferPostingRequest`**：`Kind = Customs`，`LedgerBizType = STOCK_TRANSFER`（与 §6 一致），`TransferHeaderId` = 移库主单 `Id`，`TransferBusinessCode` = 移库单号，`LineSnapshots` = 各报关行对应在库迁移。 |
| **内核职责** | **唯一** `stock`/`stock_item` 数量变更；写入 **虚拟 `stock_out`（类型3）+ 虚拟 `stock_in`（类型3）**；写 **`stockledger`**；备货可用量等 **反范式同步**（与手工移库内核路径一致）。 |
| **与手工移库差异** | 仅 **编排层与主表** 不同；**内核同一套**；**不得**把报关迁库写入 `stocktransfer_manual`。 |
| **当前代码占位** | `CompleteDeclarationAndTransferAsync` 内异常文案曾描述「扣源、虚拟入库、stockledger」— 实装时须 **同步为「+ 虚拟出库」** 并与本文 §4.4、共享内核 §四 一致。 |
